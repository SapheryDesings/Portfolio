Introducción

Un programa es un conjunto de instrucciones dadas al ordenador en un lenguaje que solo es entendible por él para comunicarle lo que queremos que haga. Un algoritmo es una secuencia finita de operaciones que resuelven un determinado problema.

Un algoritmo es un boceto de como resolver un problema. Mientras que un programa es al conjunto de tareas o instrucciones a realizar por un ordenador.

Caractéristicas de un programa:

  • Deber ser finito: formado por un conjunto limitado de sentencias.
  • Debe ser legible: es importante crear códigos limpios y fáciles de leer con tabulaciones y espacios que diferencien las partes del programa.
  • Debe ser modificable: debe ser sencillo el proceso de actualización o modificación ante nuevas necesidades.
  • Debe ser eficiente: debemos crear programas que ocupen poco espacio en memoria y se ejecuten rápidamente.
  • Debe ser modulable: debemos realizar algoritmos que se dividan a su vez en subalgoritmos de forma que se disponga de un grupo principal desde el que llamaremos al resto. Así, incitamos a la reutilización de código.
  • Debe ser estructurado: será fácil de leer y modificar, y estará compuesto de subprogramas que permitirán la reutilización de código.

Representaciones gráficas

Para representar un algoritmo, podemos utilizar dos herramientas gráficas: los diagramas de flujo y el pseudocódigo.

Un diagrama de flujo es una representación gráfica de un proceso. Cada paso del proceso se representa con un símbolo diferente que contiene una breve descripción.

Debemos usar una serie de símbolos:

  • Rectángulo esquinas redondas: Indican inicio o fin de programa.
  • Rectángulo: Representan una instrucción, un paso a dar o un proceso.
  • Paralelogramo: Operaciones de entrada/salida de datos.
  • Rombo: Usaremos este símbolo si nos encontramos en un punto en el que se realizará una u otra acción en función de una decisión.
  • Circulo: Conector. Permite unir diferentes zonas del diagrama de forma que se redefine el flujo de ejecución hacia otra parte del diagrama.
  • Flechas: para indicar la dirección de flujo.
Arriba

Características de los diagramas de flujo

  • Deben escribirse de arriba abajo o de izquierda a derecha.
  • Debemos evitar el cruce de lineas, el unos de conectores debe producirse cuadno no exista otra opción.
  • Todas las líneas de flujo deben estar conectadas a algún objeto.
  • El texto que se escribe en las formas debe ser escueto y legible.
  • Todos los símbolos de decisión deben tener más de una línea de salida, deben indicar qué camino seguir en función de la decisión tomada.

La otra herramienta para representar algoritmos es el pseudocódigo, que es la manera de representar distintas sentencias que va a realizar nuestro algoritomo. Este no se puede ejecutar en una máquina.

Características del pseudocódigo

  • Comienza con la palabra algoritmo seguido de su nombre.
  • Secuencia de intrucciones de este algoritmo.
  • La estructura finaliza con FinAlgoritmo.
  • Es recomendable la indentación, aunque no necesaria.
  • No se diferencia entre mayúsculas y minúsculas.

Comunicación de un sistema informático

En el proceso de programación se comunica a un usuario con una máquina, se necesita:

  • Usuario y máquina.
  • El canal, Ej.: Teclado
  • El lenguaje. debemos acercar los lenguajes para así lograr un entendimiento.

Para solventar el problema de la comunicación tenemos lenguajes de tres tipos, alto nivel, nivel medio y bajo nivel. Los de alto nivel están más cerca del usuario, por el contrario los de bajo nivel se acercan más al lenguaje máquina.

Implementararemos código con lenguajes de alto nivel para el cual necesitaremos traducción para convertir dicho programa en lenguaje máquina.

Arriba

Características del lenguaje de alto nivel

  • Es totalmente independiente de la máquina y muy portable.
  • Muy utilizado en el mercado laboral.
  • Modificaciones como actualizaciones son fáciles de realizar.
  • Para la traducción necesitaremos un compilador y un enlazador con librerías del lenguaje elegido.

Características del lenguaje de bajo nivel

  • Está relacionado con las características del hardware.
  • Fácilmente traducible al lenguaje máquina.
  • Es totalmente dependiente de la máquina.

La estructura de un programa informático y sus elementos principales son las variables, las constantes y los distintos operadores.

Lenguajes compilados e interpretados

Los compilados son como su nombre indica aquellos que utilizan un compilador para traducirlos al lenguaje máquina.

En los interpretados se efectúa directamente la traducción y ejecución simultáneamente para cada una de las sentencias. Un intérprete verifica cada línea cuando se escribe. La ejecución es más lenta debido a esta traducción simultánea.

Características de C#

  • Orientado a objetos
  • Dos categorías de tipos de datos: valor y referencia.
  • La migración del C# a Java es sencilla.
  • C# está inculuido en la plataforma .NET que tiene el recolector de basura del CLR (Entorno en tiempo de ejecución).
  • En C# todos los tipos derivan a una superclase llamada System. Object. Esta también se aplica a los tipos básicos.
  • Es compilado
  • Genera un código intermedio llamado MSIL y ese código se vuelve a compilar para obtener el formato nativo en la máquina.

Características de Java

  • Orientado a objetos
  • Los argumentos con siempre pasados por valor.
  • Migración sencilla de Java a C#.
  • Gracias al recolector de basura, se elimina la necesidad de la liberación explícita de memoria.
  • Soporta sincronización de múltiples hilos de ejecución al nivel de lenguaje.
  • Compilado.
  • Interpretado ya que el código máquina se puede ejecutar directamente siempre que la máquina disponga del intérprete y el sistema de ejecución en tiempo real.
