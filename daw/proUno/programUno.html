Tema 1.- Introducción a la programación

Un programa es un conjunto de instrucciones dadas al ordenador en un lenguaje que solo es entendible por él para comunicarle lo que queremos que haga. Un algoritmo es una secuencia finita de operaciones que resuelven un determinado problema.

Un algoritmo es un boceto de como resolver un problema. Mientras que un programa es al conjunto de tareas o instrucciones a realizar por un ordenador.

Caractéristicas de un programa:

  • Deber ser finito: formado por un conjunto limitado de sentencias.
  • Debe ser legible: es importante crear códigos limpios y fáciles de leer con tabulaciones y espacios que diferencien las partes del programa.
  • Debe ser modificable: debe ser sencillo el proceso de actualización o modificación ante nuevas necesidades.
  • Debe ser eficiente: debemos crear programas que ocupen poco espacio en memoria y se ejecuten rápidamente.
  • Debe ser modulable: debemos realizar algoritmos que se dividan a su vez en subalgoritmos de forma que se disponga de un grupo principal desde el que llamaremos al resto. Así, incitamos a la reutilización de código.
  • Debe ser estructurado: será fácil de leer y modificar, y estará compuesto de subprogramas que permitirán la reutilización de código.

1.1.- Representaciones gráficas

Para representar un algoritmo, podemos utilizar dos herramientas gráficas: los diagramas de flujo y el pseudocódigo.

Un diagrama de flujo es una representación gráfica de un proceso. Cada paso del proceso se representa con un símbolo diferente que contiene una breve descripción.

Debemos usar una serie de símbolos:

  • Rectángulo esquinas redondas: Indican inicio o fin de programa.
  • Rectángulo: Representan una instrucción, un paso a dar o un proceso.
  • Paralelogramo: Operaciones de entrada/salida de datos.
  • Rombo: Usaremos este símbolo si nos encontramos en un punto en el que se realizará una u otra acción en función de una decisión.
  • Circulo: Conector. Permite unir diferentes zonas del diagrama de forma que se redefine el flujo de ejecución hacia otra parte del diagrama.
  • Flechas: para indicar la dirección de flujo.
Arriba

1.2.- Características de los diagramas de flujo

  • Deben escribirse de arriba abajo o de izquierda a derecha.
  • Debemos evitar el cruce de lineas, el unos de conectores debe producirse cuadno no exista otra opción.
  • Todas las líneas de flujo deben estar conectadas a algún objeto.
  • El texto que se escribe en las formas debe ser escueto y legible.
  • Todos los símbolos de decisión deben tener más de una línea de salida, deben indicar qué camino seguir en función de la decisión tomada.

La otra herramienta para representar algoritmos es el pseudocódigo, que es la manera de representar distintas sentencias que va a realizar nuestro algoritomo. Este no se puede ejecutar en una máquina.

Arriba

1.3.- Características del pseudocódigo

  • Comienza con la palabra algoritmo seguido de su nombre.
  • Secuencia de intrucciones de este algoritmo.
  • La estructura finaliza con FinAlgoritmo.
  • Es recomendable la indentación, aunque no necesaria.
  • No se diferencia entre mayúsculas y minúsculas.

1.4.- Comunicación de un sistema informático

En el proceso de programación se comunica a un usuario con una máquina, se necesita:

  • Usuario y máquina.
  • El canal, Ej.: Teclado
  • El lenguaje. debemos acercar los lenguajes para así lograr un entendimiento.

Para solventar el problema de la comunicación tenemos lenguajes de tres tipos, alto nivel, nivel medio y bajo nivel. Los de alto nivel están más cerca del usuario, por el contrario los de bajo nivel se acercan más al lenguaje máquina.

Implementararemos código con lenguajes de alto nivel para el cual necesitaremos traducción para convertir dicho programa en lenguaje máquina.

Arriba

1.5- Características del lenguaje de alto nivel

  • Es totalmente independiente de la máquina y muy portable.
  • Muy utilizado en el mercado laboral.
  • Modificaciones como actualizaciones son fáciles de realizar.
  • Para la traducción necesitaremos un compilador y un enlazador con librerías del lenguaje elegido.
Arriba

1.6.- Características del lenguaje de bajo nivel

  • Está relacionado con las características del hardware.
  • Fácilmente traducible al lenguaje máquina.
  • Es totalmente dependiente de la máquina.

La estructura de un programa informático y sus elementos principales son las variables, las constantes y los distintos operadores.

1.7.- Lenguajes compilados e interpretados

Los compilados son como su nombre indica aquellos que utilizan un compilador para traducirlos al lenguaje máquina.

En los interpretados se efectúa directamente la traducción y ejecución simultáneamente para cada una de las sentencias. Un intérprete verifica cada línea cuando se escribe. La ejecución es más lenta debido a esta traducción simultánea.

Características de C#

  • Orientado a objetos
  • Dos categorías de tipos de datos: valor y referencia.
  • La migración del C# a Java es sencilla.
  • C# está inculuido en la plataforma .NET que tiene el recolector de basura del CLR (Entorno en tiempo de ejecución).
  • En C# todos los tipos derivan a una superclase llamada System. Object. Esta también se aplica a los tipos básicos.
  • Es compilado
  • Genera un código intermedio llamado MSIL y ese código se vuelve a compilar para obtener el formato nativo en la máquina.

Características de Java

  • Orientado a objetos
  • Los argumentos con siempre pasados por valor.
  • Migración sencilla de Java a C#.
  • Gracias al recolector de basura, se elimina la necesidad de la liberación explícita de memoria.
  • Soporta sincronización de múltiples hilos de ejecución al nivel de lenguaje.
  • Compilado.
  • Interpretado ya que el código máquina se puede ejecutar directamente siempre que la máquina disponga del intérprete y el sistema de ejecución en tiempo real.

Tema 2.- Estructura de un programa informático

Para desarrollar nuestros programas utilizaremos el lenguaje de programación C# es un lenguaje de alto nivel que pertenece al conjunto de lenguajes de .NET. Para ello utilizaremos un IDE, un entorno de desarrollo integrado diseñado para el desarrollo de aplicaciones con un conjunto de herramientas integradas en una interfaz gráfica de usuario (GUI). Sus características son:

  • Editor de código fuente: Ayuda a escribir el código resaltando la sintaxis de forma visual y rellenando de forma automática las distintas opciones que nos proporciona el lenguaje.
  • Depurador: Localiza errores de código.
  • Compilación del código fuente: Un código binario entendible por la máquina.

2.1.- Bloques de un programa informático

Un programa es una secuencia de intrucciones separadas por un punto y coma que se van a ir agrupando en diferentes bloques mediante llaves.

Cuando creamos un nuevo proyecto, se crea la siguiente estructura.

Primero nos crea la clase program. Una clase es una estructura de datos que está compuesta por atributos o variables y métodos o funciones.

El método principal se denomina Main. Será el primero en ejecutarse en nuestro programa. Este tiene una serie de argumentos, que son cadenas de caracteres que podemos pasar línea de comandos.

Entrada y salida por pantalla

Emplearemos la clase Console para establecer la comunicación entre usuario y el programa.

Para la salida por pantalla, vamos utilizar los métodos de esta clase, Write y Writeline:

  • Write() escribe un valor o valores especificados en el flujo de salida estándar.
  • WriteLine() escribe un valor o valores especificado y se realiza un salto de línea en nuestra consola.

Para la entrada de datos, utilizaremos el método ReadLine(). Este leerá el dato introducido por el usuario y devolverá un dato de tipo cadena (string).

Otro método es Read(), que leerá el dato introducido y devolverá un dato tipo entero (int).

También podemos utilizar ReadKey(), aquí se obtiene el carácter presionado por el usuario y se muestra esta tecla por la pantalla.

2.2.- Variables. Usos y Tipos

Una variable es una estructura de datos que ocupan un espacio en memoria y pueden variar a lo largo de un programa.

Cuando declaramos una variable establecemos el tipo de dato (entero, cadena, carácter, etc) y su identificador.

Tipos de datos en C#:

  • Enteros con signo: Sbyte, short, int, long.
  • Enteros sin signo: Byte, unshort, unit, ulong.
  • Punto flotante: Float, double.
  • Uno o varios caracteres: Char.
  • Decimal de alta precisión: Decimal.
  • Booleano: Bool

Tipo Sting: son cadenas de caracteres.

Tipo de dato enumerado, enum: días_semana {lunes, martes, miercoles};

Tipos matriz: Unidimensionales y Multidimensionales.

Para definir una variable, primero necesitamos conocer el tipo de datos que va almacenar y el nombre que le vamos asignar. Es importante que el nombre sea identificativo con su contenido.

Para identificar a una variable, debemos seguir una serie de normas:

  • Debe comenzar por un carácter (letra o "_").
  • Debe estar formada por caracteres del alfabeto (no podemos usar ñ) o por dígitos (del 0 al 9). Se puede utilizar el subrayado("_").
  • No debe comenzar con un dígito.
  • Distingue entre mayúsculas y minúsculas.
  • No puede llevar espacios en blanco.
  • No se pueden utilizar palabras reservadas.

A las variables también se les puede asignar un valor, ej.: int num = 5;

Definimos una variable de tipo entero, llamada num y le asignamos el valor 5.

Las variables pueden ser dependiendo de su ámbito:

  • Variables locales: aquellas declaradas dentro de un método o función.
  • Variables globales: aquellas declaradas fuera de un método o función. Podemos acceder a ellas desde cualquier punto del programa.

No es buena prática utilizar variables globales. El problema es que se crean dependencias ocultas. Cuando la aplicación es grande, no se conoce los objetos que tiene ni sus relaciones.

2.3.- Constantes y Literales. Tipos y utilidades

Constantes

Son aquellas cuyo contenido no se puede alterar a lo largo del programa. La palabra reservada es const, y se definen fuera del cuerpo de cualquier función, normalmente al principio del programa.

Literales: Así podemos representar de forma explícita todos los valores que pueden tomar los tipos básicos del lenguaje.

Literales enteros:

  • Decimales: sin ningún prefijo.
  • Hexadecimales: con el prefijo de 0x o 0X.

Literales reales: sin sufijo o con el sufijo d o D es del tipo double.Con el sufijo f o F es del tipo float. Con el sufijo m o M es del tipo decimal.

Literales de lógicos: representados por los valores lógicos true (verdadero) y false (falso).

Literales de carácter: cualquier carácter en C# podemos representarlo con comillas simpres ´´.

Existen caracteres especiales:

  • Comilla simple: \´
  • Comilla doble: \"
  • Carácter nulo: \0
  • Retroceso: \b
  • Salto de página: \f
  • Nueva línea: \n
  • Retorno de carro: \r
  • Tabulación horizontal: \t
  • Tabulación vertical: \v

2.4.- Operadores del lenguaje de programación

Operadores aritméticos

  • + Suma aritmética de dos valores.
  • ++ Incremento en 1 del operando.
  • - Resta aritmética de dos valores.
  • -- Decremento en 1 del operando.
  • * Multiplicación aritmética de dos valores.
  • / División aritmética de dos valores.
  • % Obtiene el resto de la división entera.

El operador de incremento ++ puede ser aplicado como prefijo o sufijo de una variable. Sucede lo mismo con el operador decremento --.

La prioridad de los operadores (de mayor a menor):

  1. De incremento y decremento sufijos.
  2. De incremento y decremento prefijos.
  3. De multiplicación, divsión, resta y %.
  4. De suma y resta.

Operadores booleanos

  • & AND lógico. Evalúa ambos operandos.
  • && AND lógico. Evalúa el operando derecho solo si es necesario.
  • | OR lógico. Evalúa ambos operandos.
  • || OR lógico. Evalúa el operando derecho solo si es necesario.
  • ^ XOR lógico.
  • ! Negación lógica.

El operador AND lógico & calcula el resultado de a & b es true si a y b se evalúan como true. Si no, el resultado es false.

&& también evalúa sus operandos, pero no evalúa el operando derecho si el izquierdo es false.

| calcula el resultado de a | b es true si a o b se evalúan como true. Si no, es false.

|| también evalúa sus operandos pero no evalúa el operando derecho si el izquierdo se evalúa como true.

^ es el operador exclusivo o XOR. La tabla de la verdad es:

a b a^b
T T F
T F T
F T T
F F F

El operador ! calcula la negación de su operando, si el operando es true, devolverá el valor false y viceversa.

Operadores de comparación e igualdad

Operadores de comparación
> Mayor
< Menor
>= Mayor o igual
<= Menor o igual
== Igual
!= Desigualdad
= Asignación

El operador < devuelve true si el operador izquierdo es menor que el derecho. El operador > devuelve true si el izquierdo es mayor que el derecho.

El operador <= devuelve true si el izquierdo es menor o igual que el derecho. >= devuelve true si el izquierdo es mayor o igual que el derecho.

El operador de igualdad == devuelve true si sus operandos son iguales.

El operador de desigualdad != devuelve true si sus operadores son distintos.

2.5.- Conversiones de tipos de clase

Cuando declaramos una variable, no se puede volver a declarar, ni asignar un valor de otro tipo. En ocasiones, podemos convertir el valor de una variable de un tipo a otro. Podemos tener diferentes clases de conversiones:

  • Conversiones implícitas: son aquellas que se producen cuadno el valor que se va a almacenar se puede almacenar en una variable sin truncarse ni redondearse. Por ejemplo almacenar una variable int dentro de una variable long.
  • Conversiones explícitas: cuando no hay posibilidad de realizar una conversión implícita debemos indicar al compilador que se pretende realizar una conversión que pueda producir una pérdida de datos.

Parse y TryParse

Con estos métodos podemos convertir una cadena en varios tipos numéricos: int, decimal, double...

El método Parse devuelve el número convertido. TryParse devuelve un valor booleano indicado si la conversión se realizó correctamente o no, y devuelve el número convertido.

Convert

Es una clase que convierte un tipo de datos en otro. Algunos métodos son:

Método Descripción
ToBoolean Convierte un valor en booleano.
ToChar Convierte un valor en un carácter.
ToDecimal Convierte un valor en un número decimal.
ToDouble Convierte un valor en un número de punto flotante.
ToInt16 Convierte un valor en un entero de 16 bits.
ToInt32 Convierte un valor en un entero de 32 bits.
ToInt64 Convierte un valor en un entero de 64 bits.
ToString Convierte un valor en su representación de cadena.

2.6.- Comentario al código

No debemos pasar por alto que, además de realizar programas, lo debemos hacer de la forma más optimizada y ordenada posible. Los comentarios son una herramienta disponible en el compilador para que se puedan hacer anotaciones en el código sin que sea procesado por el compilador. Está permitido hacer dos tipos de comentarios:

  • Comentarios de una línea: son frases cortas, solo pueden ocupar una línea del código. Para comenzar escribir // antes de comenzar con dichas anotaciones.
  • Comentarios multiline o multilíneas: Son comentarios de varias líneas, para una explicación más detallada. Debemos englobar el contenido entre los caracteres /* y */.

El uso de comentarios es muy útil, se aconseja su uso para documentar correctamente el código.
