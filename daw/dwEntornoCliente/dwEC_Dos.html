2.1.- Selección de lenguaje de programación de clientes web

Estudiaremos el lenguaje JavaScript, Jscript, es una variante de JavaScript pero desarrollada por otra empresa(Microsoft).

El lenguaje JavaScript toma como referencia el lenguaje C. Los lenguajes de la programación moderna de alto nivel parten de la misma base. Esto facilita la adaptación, la conversión y la compatibilización de JavaScript con lenguajes de la misma base.

La sintaxis de las sentencias tiene un formato parecido.

JavaScript fue desarrollado por la empresa Netscape, con el nombre inicial de Mocha. Seguidamente pasó a llamarse LiveScript.

Pertenece actualmente a la empresa Oracle Corporation (propietaria también de Java), aunque usa licencias de productos Netscape.

2.2.- Tipos de datos

En JavaScript no es necesario almacenar el mismo tipo de datos en una misma variable, ya que en su declaración no se especifica el tipo.

Los tres tipos de datos primitivos son los números, las cadenas de texto y los valores booleanos. Existen otro tipo de datos compuesto llamado Object.

String Caracteres de texto
Number Tipo numérico(decimal o entero)
Boolean Es una entidad lógica que puede tener dos valores: true o false.
Null Tiene exactamente el valor: null
Object Variable que pertenece a un objeto de una clase
Undefined Variable que no han sido inicializadas

Conversiones de tipos de datos

Existe la posibilidad de convertir los tipos de datos en el tipo de dato deseado por el desarrollador.

Las funciones para la conversión de cadenas de caracteres en datos numéricos son: parseInt() y parseFloat().

El proceso contrario es mucho más sencillo, ya que lo ejecuta el compilador al detectar las cadenas de caracteres como tipo por defecto. Pasar un tipo entero a cadena de caracteres, lo único que debemos hacer es concatenar una cadena vacía al principio de la variable.

2.3.- Variables

Definimos variable como una zona de memoria identificada con un nombre y en la cual se almacenan ciertos datos.

En el momento en que el navegador limpia la ventana, cualquier variable conocida será eliminada.

El primer paso para hacer uso de una variable es la creación o declaración en el código. Dos formas:

La palabra reservada var seguida del nombre de la variable o identificador.

                
                    var numero;

También podemos asignar un valor durante la operación.

                
                    var numero = 14;

En la declaración de la variable no hay que indicar el tipo de datos que va a contener. Una variable de JavaScript podrá almacenar diferentes tipos de valores.

Para identificar una variable:

  • El nombre que identifica una variable debe estar relacionado con el dato que almacena.
  • No se pueden utilizar palabras reservadas por el compilador del lenguaje para identificar la variable.
  • El nombre de una variable no puede contener espacios en blanco ni símbolos de puntuación, solo cacacteres alfanuméricos y el guión bajo( _ ).
  • La identificación de una variable no puede comenzar por un número.

2.4.- Asignaciones

Esta operación almacena el valor en la variable correspondiente. Se lleva a cabo mediante el operador (=).

                
                    // Operaciones de asignación.
                    var a = 1;
                    var z = 10;
                    z = a;
                    //Primero se incrementa el valor de a. A continuación,
                    se asigna a z.
                    z = ++a;
                    //Primero se asigna el valor de a (2) a z. A continuación,
                    se incrementa a (3).
                    z = a++;

Los operadores para realizar la operación de asignación son: = += -= *= /= %= <<= >= >>= >>>= &= |= ˆ= []

Asigna el valor a la derecha de la expresión a la variable que está a la izquierda.

2.5.- Operadores

Las operaciones que se pueden realizar con ellas. Estas operaciones se llevan a cabo mediante los operadores.

Cualquier variable o valor sobre el que se realiza una operación se denomina operando.

Presentamos los distintos tipos de operaciones:

2.5.1. - Comparación

Podemos comparar los valores de los dos operandos que intervienen en la operación, devolviendo una condición como resultado (true o false).

Operador Nombre Descripción
< Menor que Verifica si el operando a la izquierda del operador es menor que el operando de la derecha. Devuelve true en ese caso o false en caso contrario
<= Menor o igual que Verifica si el operando a la izquierda del operador es menor o igual que el operando de la derecha. Devuelve true en ese caso o false en caso contrario
== Igual Verifica si los dos operandos son iguales. Devuelve true en ese caso o false en caso contrario
> Mayor que Verifica si el operando a la izquierda del operador es mayor que el operando de la derecha. Devuelve true en ese caso o false en caso contrario
>= Mayor o igual que Verifica si el operando a la izquierda del operador es mayor o igual que el operando de la derecha. Devuelve true en ese caso o false en caso contrario
!= Diferente Verifica si los dos operandos son diferentes. Devuelve true en ese caso o false en caso contrario
=== Estrictamente igual Verifica si el operando a la izquierda del operador es igual y del mismo tipo de datos que el operando de la derecha. Devuelve true en ese caso o false en caso contrario
!== Estrictamente diferente Verifica si el operando a la izquierda del operador es diferente y/o de tipo diferente que el operando de la derecha. Devuelve true en ese caso o false en caso contrario

2.5.2.- Aritméticos

Realiza las operaciones de suma, resta, multiplicación, división y resto de la división entre operandos. Devuelven como resultado un tipo numérico.

Operador Nombre Descripción
+ Suma Efectúa la suma entre dos operandos
- Resta Efectúa la resta entre dos operandos
* Multiplicación Efectúa la multiplicación entre dos operandos
/ División Efectúa la división entre dos operandos
% Módulo Extrae el resto del resultado de la división entre los operandos
++ Incremento Permite incrementar un valor
-- Decremento Permite decrementar un valor

Booleano

Operaciones de conjunción entre los operandos. Devuelve una condición a partir de dos condiciones. Lógicos tienen la utilidad de combinar o manipular diferentes expresiones lógicas, con el fin de evaluar si el resultado de esta combinación es verdadero o falso.

Operador Nombre Descripción
&& Y Ejecuta la operación booleana AND sobre los valores. Devuelve true solo si todos los valores son true. Devuelve false en caso contrario
|| O Ejecuta la operación booleana OR sobre los valores. Devuelve true en el caso de que al menos uno de los valores sea true. Devuelve false en caso contrario
! No Invierte el valor booleano de su operando

2.5.4.- Asignación

Además del operador de asignación(=), cuenta con otros operadores de asignación que tienen una característica común, ahorrarnos tiempo y disminuir la cantidad de código escrito. Evitaremos escribir dos veces el nombre de la variable.

Operador Nombre Descripción
+= Suma y asigna Ejecuta una suma y asigna el valor al operando de la izquierda
-= Resta y asigna Ejecuta una resta y asigna el valor al operando de la izquierda
*= Multiplica y asigna Ejecuta una multiplicación y asigna el valor al operando de la izquierda
/= Divide y asigna Ejecuta una división y asigna el valor al operando de la izquierda
%= Módulo y asigna Ejecuta el módulo y asigna el valor al operando de la izquierda

2.5.5.- Condicionales

Son un tipo de operadores algo más complejos, pero bastante útiles a la hora de tomar decisiones, podemos indicarle al navegador que ejecute una acción en concreto después de evaluar una expresión.

Consta de tres partes: la primera es la expresión que evaluar, la segunda es la acción que realizar si la expresión es verdadera y la tercera es la acción que ejecutar si la expresión es falsa.

Operador Nombre Descripción
?: Condicional Si la expresión antes del operador es verdadera, se utiliza el primer valor a la derecha. En caso contrario, se utiliza el segundo valor a la derecha. 
                
                    //Sintaxis
                    condición ? valor1 : valor2
                    //Ejemplo
                    var estado = (edad >= 18) ? “adulto” : “menor”;

2.6.- Sentencias

Las aplicaciones están compuestas por sentencias con sintaxis propia. Puede formarse por múltiples líneas y contener varias sentencias en una sola línea si están separadas por un punto y coma. No es una sola palabra clave, sino un grupo de palabras clave.

Control de flujo

block Un bloque de sentencias es utilizado para agrupar sentencias. El bloque está delimitado por llaves
break Finaliza una sentencia loop, switch, o label y transfiere el control a la siguiente sentencia
continue Finaliza la ejecución de la sentencias en el interior del actual bucle y continúa la ejecución del bucle con la siguiente iteración
empty Una sentencia vacía se utiliza para proveer una "no sentencia", aunque la sintaxis de JavaScript esperaba una
if...else Si la condición indicada es true, ejecutará la sentencia, en cambio, si la condición es false, es posible ejecutar otra sentencia diferente.
switch Evalúa una expresión, igualando el valor obtenido en la expresión a una cláusula case y ejecutando las sentencias pertinentes
throw Lanza una excepción predefinida por el usuario
try...catch Prueba un bloque se sentencias (try) y especifica una respuesta en caso de lanzarse una excepción

Declaraciones

var Declara una variable
let Declara una variable local de ámbito de bloque
const Declara una constante solamente de lectura

Funciones

function Declara una función con unos parámetros especificados
function* Los generadores de funciones permiten escribir iteradores de una manera más fácil
return Especifica el valor que será devuelto por una función
class Declara una clase

Iteraciones

do...while Crea un bucle que ejecutará una instrucción especifica hasta que la condición se evalúe como falsa. La condición es evaluada después de ejecutar la instrucción, por lo que, como mínimo, la instrucción especificada se ejecuta una vez
for Crea un bucle con tres expresiones opcionales, situadas entre paréntesis y separadas por punto y coma. Mientras se cumpla la condición, se ejecutarán las condiciones
for...in Itera sobre las propiedades numerables de un objeto, en orden arbitrario. Para cada propiedad distinta, las instrucciones pueden ser ejecutadas
while Crea un bucle que ejecutará la instrucción especificada siempre y cuando la condición de prueba se evalúe como verdadera. La condición es evaluada antes de ejecutar la instrucción

Otros

debugger Invoca una funcionalidad de depuración disponible. Si no hay ninguna disponible, esta instrucción no tendrá efecto
export Se utiliza para permitir a un script implementar propiedades, funciones y objetos a otros scripts
import Usada para permitir a un script obtener propiedades, funciones y objetos desde otro script
label Proporciona una instrucción con un identificador que se puede referir usando una instrucción break o continue
with Extiende la cadena de alcance para una instrucción

2.7.- Estructuras de control

Instrucciones que hemos implementado se ejecutan de forma secuencial, si leemos el programa de arriba abajo y de izquierda a derecha, podemos saber qué está haciendo el programa. Hay tres tipos de sentencias condicionales:

  • if
  • switch
  • while y for

Sentencia if

Condicional Simple

Tras evaluar una expresión lógica, ejecuta una serie de instrucciones siempre y cuando la expresión lógica sea verdadera. Si la expresión no es verdadera, no ejecutará dichas instrucciones.

                
                    if (expresión lógica) {
                        Instrucciones
                        ...
                    }

Las llaves de apertura y cierre de la sentencia {y} se utilizan cuando tenermos más de una instrucción. En otro caso, es posible crear la sentencia sin llaves.

                
                    if (expresión lógica) instrucción;

Ejemplo

                
                    if (nota >= 5) {
                        console.log(“Aprobado”)
                        alumnos_aprobados++;
                    }
Condicional compuesta

Se añade un apartado else que contiene instrucciones que se ejecutarán si la instrucción no es verdadera.

                
                    if (nota >= 5) {
                        console.log(“Aprobado”)
                        alumnos_aprobados++;
                    }
                    else {
                        console.log(“Suspenso”)
                        alumnos_suspendidos++;
                    }
Anidación

Dentro de las sentencias if es posible colocar otra sentencia if. Esto es conocido como anidación, puede ir en el interior del if o del else.

Esta sintaxis es conocida como if-else-if

                
                    if (x==1) {
                        Instrucciones
                    }
                    else if (x==2) {
                        Instrucciones
                    }
                    else if (x==3) {
                        Instrucciones
                    }
                    else {
                        Instrucciones
                    }

2.7.2.- Sentencia Switch

En ocasiones, es necesario comparar el valor de una variable. Tiene una expresión que evaluar y diferentes posibles valores de dicha expresión, conocidos como casos, en los que se incluyen una serie de instrucciones que se ejecutarán cuando coincida el valor de la expresión y el valor del caso.

                
                    switch (expresión) {
                    case valor1:
                        Instrucciones
                        break;
                    case valor2:
                        Instrucciones
                        break;
                    case valor3:
                        Instrucciones
                        break;
                    default:
                        Instrucciones
                    }

Cada uno de los diferentes casos debe finalizar con la palabra reservada break. Esta palabra detiene la ejecución de la sentencia switch en el momento que uno de los casos es verdadero. No se pierde el tiempo evaluando el resto de los casos.

En el interior de la sentencia default se ejecutarán cuando ninguno de los casos haya coincidido con el valor de la expresión.

2.7.3.- Bucle For

Este tipo de bucle se repartirá un bloque de instrucciones un número limitado de veces. Permite al navegador ejecutar las instrucciones que se encuentran en el interior de la sentencia hasta que la expresión condicional devuelva el valor de false.

                
                    for (expresión inicial; condición; incremento)
                    {
                    //Instrucciones a ejecutar dentro del bucle.
                    }

2.7.4.- Bucle While

En este tipo de bucles no podemos gestionar la cantidad de veces que se pueden ejecutar, pero disponemos de una serie de bloques de ejecución para repetirse un número definido de veces, siempre que se cumpla una condición.

                
                    while (condición)
                    {
                    //Instrucciones a ejecutar dentro del bucle.
                    }

Tipo de bucle do...while y es una variación del bucle while(), se utiliza cuando no sabemos el número de veces que se habrá de ejecutar el bucle. La diferencia es que sabemos con seguridad que este se ejecutará al menos una vez.

                
                    do {
                        //Instrucciones a ejecutar dentro del bucle.
                    }while (condición);

Ejemplo

                
                    //Ejemplo de un bucle FOR
                    for (var i = 1; i <= 40; i++)
                    {
                    //Instrucciones que se ejecutarán 40 veces.
                    console.log(i);
                    }
                    //Ejemplo de un bucle WHILE
                    var i = 0;
                    while (i <= 10)
                    {
                    //Instrucciones a ejecutar dentro del bucle
                    hasta que i sea mayor que 10 y no se cumpla la
                    condición.
                    console.log(i);
                    i++;
                    }
                    //Ejemplo de un bucle DO-WHILE
                    var a = 1;
                    do{
                    //Mostrará por consola 2 veces.
                    console.log(“El valor de a es: “ + a);
                    a++;
                    }while (a < 3);

Comentarios al código

Los comentarios son sentencias que el compilador del lenguaje no tiene en cuenta. Permiten a los programadores dejar anotaciones sobre las diferentes decisiones que se han ido tomando a la hora de desarrollar el código. Ocupan espacio dentro de tu código y forman parte permanente en el código para que siempre se puedan consultar.

JavaScript tiene disponible dos tipos de comentarios. En el caso de que ocupen una sola línea, empezamos el comentario //, para tener un comentario multilínea, que va a ocupar varias líneas, debe ir entre los caracteres /* y */.

                
                    //Comentario de una sola línea
                    /*
                    Comentarios de varias líneas
                    1.
                    2.
                    */

Herramientas para programar, probar y depurar el código

La herramienta para poder implementar el lenguaje JavaScript puede ser un editor cualquiera.

Características que nos ofrecen estos editores:

  • Colorear el código: Al distinguir las palabras clave y los valores de una variable en diferentes colores, facilita la escritura/lectura del código. Es posible modificar estos temas y predefinir nuestras preferencias.
  • Facilitar la selección del texto:Existen combinaciones de teclado que nos facilitan la selección del texto y por otro lado distinguen las partes del código o texto normal.
  • Navegación avanzada: En documentos con cientos de líneas de código, nos permiten agrupar o expandir bloques de código, miniaturas para acceder más rápido a la zona que nos interesa, ir directamente a una línea de código específica.
  • Búsqueda y reemplazo: Operaciones fundamentales, buscar y modificar un texto entre diferentes archivos. Podemos modificar el nombre de una variable en todos los archivos en lo que aparezca de una manera rápida y cómoda.
  • Autocorrección a la hora de escribir: Se han incorporado correctores que indiquen la palabra incorrecta.
  • Uso de abreviaturas: Son trozos de código que se utilizan y se asocian a una abreviatura. Si escribimos "cl", al pulsar el tabulador nos lo cambia por "console.log".
  • Añadir extensiones o plugins: Permiten añadir funcionalidades que normalmente no están incluidas en un editor.

2.9.1.- Tipo de editores

  • Editores de texto multipropósito: Estos editores están especializados en facilitar la escritura en código en cualquier lenguaje. Son capaces de adaptarse al código que se está introduciendo.

    Son editores rápidos y ligeros. Ofrecen la posibilidad de incorporar plugins y extensiones para incorporar nuevas herramientas.

    Sublime Text, Notepad++, Geany y TexMate.

  • Editores específicos para el desarrollo web: La diferencia es que ya vienen preparados con herramientas y elementos para facilitarnos el desarrollo del código de las aplicaciones web (HTML, CSS, JavaScript).

    Visual Studio Code, Atom, CodeKit y Brackets.

  • Entornos de desarrollo integrados(IDE): Son softwares más completos e incorporan todas las herramientas que se necesitan. Son entornos de trabajo más pesados que los anteriores, tardan en ejecutarse y normalmente son de pago.

    WebStorm, NetBeans, Komodo y VisualStudio.

  • Editores WYSIWYG: Es el acrónimo de "lo que ves es lo que obtienes". No son ideales.

    Adobe Dreamweaver, SeaMonkey, Coffee Cup y Mobirise.

  • Editores online: Son editores que permiten escribir y visualizar el contenido al instante. Su ventaja es que no requieren una instalación en nuestra máquina para comprobar su funcionamiento.

    CodePen, Webmakerapp o CSS Deck.
