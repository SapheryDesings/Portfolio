Tema 4. - Programación con arrays, funciones y objetos definidos por el usuario

Variable capaz de gestionar conjuntos de datos. Son los arrays de datos, también conocidos como listas, vectores o matrices.

Los arrays son variables que nos permiten almacenar en una misma estructura un conjunto de valores. Para poder acceder a un dato dentro del array es necesario indicar su posición, que es un número entero conocido como índice.

El primer elemento del array tiene índice 0.

Declaración e incicialización de arrays

Para crear los arrays:

                
                    var marca = []; //Declarar un array vacío
                    var marca = new Array(); //Declarar un array con
                    el operador new con longitud 0
                    var marca = new Array(100); //Declarar un array
                    con el operador new de tamaño 100
                    //Colocar un valor en la primera posición
                    marca[0] = “Seat”;

Para seguir añadiendo valores:

                
                    marca[1] = “BMW”;
                    marca[2] = “Audi”;
                    marca[3] = “Ford”;

Si queremos mostrar un elemento en concreto de la lista:

                
                    console.log(marca[1]); //Muestra: BMW

Añadir valores en la misma declaración:

                
                    var marca = [“Seat”, “BMW”, “Audi”, “Ford”];

Hacer la inicialiazción con el operador new:

                
                    var marca = new Array(“Seat”, “BMW”, “Audi”,
                    “Ford”);

Uso de los arrays mediante bucles

Las ventajas que supone trabajar con arrays, podemos mezclar los bucles y los arrays, de esta forma podemos trabajar con muchas variables.

Si pretendemos crear un array que almacene códigos de determinados productos:

                
                    var codigos = new Array();
                    for (var i = 0; i < 10; i++) {
                    codigos[i] = “Código” + i;
                    }

Utilizará la variable i para acceder a cada uno de los índices del array y asignarle un valor.

Aparte de la inserción de valores en un array, también nos aporta mostrar los datos o trabajar sobre ellos.

Al utilizar el bucle for evitamos escribir una a una las líneas de código y optimizar el código:

                
                    for (var i = 0; i < 10; i++) {
                        document.write( codigos[i] + “< br >);
                    }

Este objeto tiene definidas una serie de propiedades y métodos. Tiene dos propiedades:

  • lenght: devuelve el número de elementos que contiene el array.
  • prototype: con esta propiedad es posible agregar nuevas propiedades y métodos al objeto array.
                
                    nombreArray.lenght;

4.1.- Funciones predefinidas del lenguaje

Dispone de una serie de funciones predefinidas, tiene funciones que están integradas en el propio lenguaje. Las podemos utilizar sin necesidad de declararlas ni conocer todas sus funcionalidades. Para el uso de estas solo tenemos que saber el nombre de la función y el resultado que vamos a obtener.

Muchas funciones pueden realizar su tarea sin necesidad de información extra.

La mayor parte de estas funciones requerirán acceder a los valores de las variables para poder producir sus resultados, estas variables que reciben las funciones se denominan argumentos o parámetros de la función.

Función predefinida Descripción
escape() Recibe una cadena de caracteres como argumento y devuelve esa misma cadena sustituida con codificación de caracteres ASCII
eval() Convierte una cadena que pasamos como argumento en código JavaScript ejecutable
isFinite() Verifica si el número que pasamos como argumento es o no un número finito
inNaN() Comprueba si el valor que pasamos como argumento es de tipo numérico
Number() Convierte el objeto pasado como argumento en un número que represente el valor de dicho argumento
String() Convierte el objeto pasado como argumento en una cadena que represente el valor de dicho objeto
parseInt() Convierte la cadena que pasamos como argumento en un valor numérico de tipo entero
parseFloat() Convierte la cadena que pasamos como argumento en un valor numérico de tipo flotante

escape()

Unescape() es la función opuesta a escape(), es decir, esta función decodifica las caracteres que estén codificados.

                
                    var input = prompt(“Introduce una cadena de texto:
                    “);
                    var inputCodigo = escape(input);
                    alert(“Cadena codificada: “) + inputCodigo;

eval()

Este ejemplo permite ingresar al usuario una operación numérica y muestra el resultado de esta.

                
                    var input = prompt(“Introduce una operación numérica:
                    “);
                    var resultado = eval(input);
                    alert(“El resultado de la operación es: “ + resultado);

isFinite()

Comprueba si el valor pasado como argumento corresponde a un número finito.

Si el argumento no se encuentra en el rango o no es un valor numérico, la función devuelve false, en caso contrario, devuelve true.

                
                    if(isFinite(argumento)){
                        //instrucciones si el argumento es un número
                        finito
                        }else{
                        //instrucciones si el argumento no es un número
                        finito
                        }

isNaN()

Comprueba si el valor pasado como argumento es numérico. Si no es numérico, devuelve false, de lo contrario, devuelve true.

                
                    var input = prompt(“Introduce un valor numérico:
                    “);
                    if(isNaN(input)){
                    alert(“El dato introducido es numérico”);
                    }else{
                    alert(“El dato introducido no es numérico”);
                    }

Number()

Comprueba si el valor pasado como argumento es numérico. Si no es numérico, devuelve false, en caso contrario, devuelve true.

                
                    var input = prompt(“Introduce un valor:”);
                    var inputNumber = Number(input);
                    alert(“El valor introducido: “ + inputNumber);

String()

String() convierte el objeto pasado como parámetro en una cadena de texto.

                
                    var fecha = new Date();
                    var fechaString = String(fecha);
                    alert(“La fecha actual es: “ + fechaString);

parseInt()

Intenta convertir una cadena de caracteres pasada como argumento en número entero con una base especificada. Si no especificamos la base, se utiliza automáticamente la base decimal(10).

La base puede ser binaria (2), octal (8), decimal (10), o hexadecimal (16).

Si la función encuentra un valor que no sea numérico, devuelve el valor encontrado hasta ese punto; si el primer valor no es numérico, devuelve NaN.

                
                    var input = prompt(“Introduce un valor: “);
                    var inputParsed = parseInt(input);
                    alert(“parseInt (“ + input + “): “ + inputParsed);

parseFloat()

Similar a la anterior, en lugar de convertir el argumento a un número entero, lo convierte en un número flotante.

Si encuentra un valor que no sea numérico, devuelve el valor encontrado hasta ese punto; si el primer valor no es numérico, devuelve NaN.

                
                    var input = prompt(“Introduce un valor: “);
                    var inputParsed = parseFloat(input);
                    alert(“parseFloat (“ + input + “): “ + inputParsed);

4.2.- Definición y llamadas de funciones

Podemos definir las funciones como un conjunto de instrucciones relacionadas para realizar una tarea específica.

Es conveniente, diseñar funciones que podamos reutilizar en otras aplicaciones. Las funciones se convertirán en pequeños bloques constructivos que permitirán desarrollar los nuevos programas de una forma más rápida.

En algunos lenguajes, se utiliza mucho el término procedimiento o subrutina, en JavaScript no vamos a diferenciar entre procedimientos (los que ejecutan acciones y devuelven algún valor), ya que siempre se van a definir como funciones mediante la palabra reservada function.

                
                    Sintaxis
                    function nombreFuncion([parametro1, parametro2, …
                    parametroN])
                    {
                    //desarrollo del código
                    }

Una función siempre debe ir desarrollada entre los símbolos {}.

Para invocar a la función, lo hacemos a través de su identificador seguido de los parámetros de entrada separados por comas y entre paréntesis. De no tener parámetros de entrada, también debe llevar los paréntesis a la hora de realizar su llamada.

                
                    //Función JavaScript
                    function miPrimeraFuncion(){
                    return “Mi primera función con Ilerna Online”;
                    }
                    //Llamada a la función
                    alert(miPrimeraFuncion());

También podemos llamar a una función dentro de una etiqueta de HTML. Debe ir acotado entre las etiquetas script.

Las funciones siempre van a devolver una respuesta mediante el comando return. En el supuesto de que devuelva un valor no especificado, devolverá undefined.

En una función, una vez llegemos al return, este devolverá un determinado valor, si existe código detrás del return, no se va a ejecutar nunca.

                
                    //Función JavaScript
                    function miPrimeraFuncion(){
                    return “Mi primera función con Ilerna Online”;
                    //Este código no se va a ejecutar nunca
                    var miVariable = 5;
                    var resultado = 3 + miVariable;
                    alert(resultado);
                    }

4.2.1. Argumentos

No existe ninguna regla que determine el número de parámetros que se debe representar. Una función espera tres argumentos: tenemos la posibilidad de poder facilitarle uno, cinco o incluso más, ya que los parámetros que recibe una función se van a gestionar de manera interna como si se tratara de un array.

En JavaScript, los parámetros siempre se pasan por valor, es decir, crea una copia en memoria del valor que recibe o del conjunto de valores. Esta forma de pasar los parámetros resulta confusa, ya que, cuando el parámetro recibe un dato primitivo o un tipo de referencia, se va a comportar de manera diferente. Es conveniente tener un paso por valor y otro por referencia, aunque JavaScript solo los realice por valor.

Tipo Valor Tipo de dato Comportamiento frente a cambios
Valores por valor Primitivo
  • Copia en una variable local del parámetro recibido
  • Apunta a una dirección de memoria distinta
  • Los cambios en el interior de la función no se trasladan fuera de esta
Valores por referencia Referencia
  • Copia en una variable local del parámetro recibido
  • Apunta a la misma dirección de memoria
  • Los cambios en el interior de la función sí se trasladan fuera de esta
 
                
                    function miPrimeraFuncion(texto){
                        return texto + “ con Ilerna Online”;
                    }
                    alert(miPrimeraFuncion(“Mi primera función”));

4.3.- Objetos definidos por el usuario

Es muy útil poder crear nuevos objetos definidos por el usuario, son sus propios métodos y propiedades.

Declaración e inicialización de objetos

Un objeto lo podríamos definir como una entidad que posee unas propiedades y unos métodos que actúan sobre estas propiedades.

Para declararlos:

                
                    function mi_objeto (valor1, valor2, valorx) {
                        this.propiedad1 = valor1;
                        this.propiedad2 = valor2;
                        this.propiedadx = valorx;
                    }

Se crea del mismo modo que creamos una función, empleando la palabra clase function. Es posible dar un nombre a este nuevo objeto y asignarle unos valores iniciales a cada una de sus propiedades definidas.

                
                    function Coche (marca_in, modelo_in, año_in) {
                        this.marca = marca_in;
                        this.modelo = modelo_in;
                        this.anyo = año_in;
                    }

Declarado el objeto Coche, es posible crear instancias de este a través de la palabra clave new.

                
                    var coches = new Array(3);
                    coches[0] = new Coche(“Seat”, “León”, “2018”);
                    coches[1] = new Coche(“Ford”, “Focus”, “2019”);
                    coches[2] = new Coche(“Hyundai”, “i30”, “2019”);

Definición de propiedades y métodos

A la hora de la creación de sus propiedades utilizamos la palabra clave this seguida del nombre de la propiedad y su valor. Esta palabra hace referencia al objeto en el que utilizamos esa palabra. Sin embargo, es posible crear nuevas propiedades sin que estas hayan sido declaradas en el mismo objeto.

Todas las instancias a este objeto deben contener estas propiedades. Es posible añadir nuevas propiedades a estos objetos.

                
                    function mi_objeto (valor1, valor2, valorx) {
                        this.propiedad1 = valor1;
                        this.propiedad2 = valor2;
                        this.propiedadx = valorx;
                    }
                    var mi_coche1 = new Coche(“Volkswagen”, “Golf”,
                    “2007”);
                    mi_coche1.color = “Gris”;

La instancia mi_coche1 tendrá una nueva propiedad que nos informará del color.

Otra de las características que debemos tener en cuenta es la posibilidad de establecer objetos como propiedad de otro objeto. Los objetos Coche que hemos creado pueden pertenecer a otro objeto que llamaremos Concesionario. Este último objeto tendrá sus propias características, como pueden ser el código de oficina y la ciudad.

                
                    function Concesionario (cod_oficina_in, ciudad_in) {
                        this.cod_oficina= cod_oficina_in;
                        this.ciudad = ciudad_in;
                    }
                    var concesionario1 = new Concesionario(“0250405”,
                    “Lleida”);
                    var mi_coche1 = new Coche(“Volkswagen”, “Golf”,
                    “2007”, concesionario1);

Para poder mostrar el contenido del objeto Coche, se creará una función que imprima por pantalla los datos.

                
                    function imprimirDatos() {
                        document.write(“< p> Marca: “ + this.marca +
                        “< /p>”);
                        document.write(“< p> Modelo: “ + this.modelo +
                        “< /p>”);
                        document.write(“< p> Año: “ + this.anyo + “”);
                    }
                    function Coche (marca_in, modelo_in, año_in) {
                        this.marca = marca_in;
                        this.modelo = modelo_in;
                        this.anyo = año_in;
                        this.imprimirDatos = imprimirDatos;
                    }
                    var mi_coche1 = new Coche(“Volkswagen”, “Golf”,
                    “2007”);
                    mi_coche1.imprimirDatos();
