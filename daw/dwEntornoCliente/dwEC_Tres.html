Tema 3. - Identificación y aplicación de los objetos predefinidos del lenguaje

Mediante el modelo de objetos del documento(DOM) es posible visualizar el documento de una forma diferente, ya que define el contenido de un determinado documento como un conjunto de objetos sobre los que JavaScript puede interactuar.

El modelo de objetos del documento(DOM) como una interfaz de programación de aplicaciones (API) para aquellos documentos HTML válidos y para los XML que estén bien construidos. Además, el DOM puede definir la estrucutra lógica de estos documentos junto con la forma para poder acceder a ellos y manipularlos.

Un objeto es una colección de propiedades que, para definir un estado, van a utilizar una serie de funciones o métodos que harán funcionar a esas propiedades.

Para la creación de un objeto se utiliza la palabra clave new.

                
                    var nombreobjeto = new Object();

Para poder acceder a las propiedades de un objeto, utilizaremos el punto(.) y el nombre de la propiedad a la cual queremos acceder:

                
                    nombreobjeto.propiedad

En cambio, para acceder a un método concreto de un objeto:

                
                    nombreobjeto.metodo([parámetros ocionales])

Cuando necesitaremos referenciar alguna propiedad por su índice, siempre comienza por 0.

3.1.3.- El objeto Number

Poco utilizado, Number dispone de una información utilizada por programadores más específicos.

Las propiedades del objeto Number se refieren al rango de valores de números que soporta el propio lenguaje, siendo el más alto el 1.79E + 308 y el más bajo, 2.22E - 308.

Los números más altos que el número mayor se considera como infinito positivo, y los más pequeños que el menor, como infinito negativo.

Todos los números junto con sus correspondientes valores se definen como valores de doble precisión de 64 bits.

3.3.- Clases

El lenguaje JavaScript está basado en prototipos y cada uno de sus objetos tiene una propiedad interna oculta nombrada [[Prototype]] que puede tener propiedades y métodos de objetos.

Las clases surgen como una mejora sobre la herencia basada en estos prototipos. No introducen un nuevo modelo, sino que aportan una sintaxis mucho más simple y clara para crear objetos y lidiar con la herencia.

Las clases son funciones especiales. La sintaxis de la clase tiene dos componentes:

  • Expresiones de clases
  • Declaraciones de clases

Para la declaración, utilizaremos la palabra reservada class junto a un nombre.

                
                    class Alumno {
                        constructor(nombre, apellido, edad) {
                            this.nombre = nombre;
                            this.apellido = apellido;
                            this.edad = edad;
                        }
                    }

Una diferencia entre declarar funciones y clases es que las declaraciones de funciones son izadas y las declaraciones de clases no lo son, tenemos que declarar la clase antes de realizar una llamada para crear objetos de esta.

                
                    var a = new Alumno (); // ReferenceError
                    class Alumno {}

También podemos definir las clases con una expresión de clase. Pueden ser nombradas o anónimas. El nombre dado a la expresión de clase nombrada es local dentro del cuerpo de esta.

                
                    // Anónima
                    var Alumno = class {
                        constructor(nombre, apellido, edad) {
                            this.nombre = nombre;
                            this.apellido = apellido;
                            this.edad = edad;
                        }
                    };
                    // Nombrada
                    var Alumno = class Alumno {
                        constructor(nombre, apellido, edad) {
                            this.nombre = nombre;
                            this.apellido = apellido;
                            this.edad = edad;
                        }
                    };

El cuerpo de una clase es la parte ubicada entre las llaves {}, donde se definen los miembros de clase, los métodos o contructores y todas las funcionalidades que queremos que realice la clase.

El método constructor es un método especial utilizado para crear e inicializar un objeto creado con una clase. Solamente puede haber un método especial con el nombre constructor en una clase. Si una clase contiene más de un método constructor, se lanzará un error SyntaxError.

Un constructor, para llamar al constructor de su superclase, utilizará la palabra super.

                
                    class Alumno {
                        constructor(nombre, apellido, edad) {
                            this.nombre = nombre;
                            this.apellido = apellido;
                            this.edad = edad;
                        }

                        get Nombre() {
                            return this.NombreCompleto();
                        }

                        NombreCompleto() {
                            return this.nombre + ‘ ‘ + this.apellido;
                        }
                    }
                    
                    function MostrarAlumno(){
                        const alumno = new Alumno(‘Ilerna’, ‘Online’,
                        23);
                        alert(‘El nombre completo del alumno es: ‘
                        + alumno.Nombre);
                    }

Es posible llamar a los métodos estáticos sin instanciar la clase. Para definir un método como estático, utilizaremos la palabra clave static. Los métodos estáticos se utilizan a menudo para crear funciones de utilidad en una aplicación.

                
                    class Calculo {
                        static multiplicacion(a, b) {
                            return a * b;
                        }
                    }
                    function MostrarMult(){
                        alert(Calculo.multiplicacion(5,10));
                    }

Para crear clases hijas de una clase genérica que tengamos (clase padre), utilizaremos la palabra clave extends.

Las clases no pueden extender objetos regulares (literales). Si se quiere heredar de un objeto regular, se debe usar Object.setPrototypeOf()

El objeto Array es un objeto global que es usado en la construcción de arrays, estos son objetos tipo lista de alto nivel.

Si queremos devolver objetos Array derivados de la clase array MyArray, lo haremos con el patrón species, el cual nos permite sobrescribir constructores por defecto.

Cuando se usan métodos de tipo map() los cuales devuelven el constructor por defecto, devolverán el objeto padre Array en vez de MyArray.

                
                    class MyArray extends Array {
                        //Sobrescribe species sobre el constructor padre
                        Array
                        static get [Symbol.species]() { return Array; }
                        }
                        function mostrarArray(){
                        var array = new MyArray(1,2,3);
                        var mapa = array.map(x => x * x);
                        alert(mapa instanceof MyArray);
                        alert(mapa instanceof Array);
                        }

El primer alert nos devolverá false, ya que la variable mapa devolverá array, como comprobamos en el segundo alert, el cual nos devuelve true.

3.4.- Creación de nuevas ventanas

El elemento u objeto ventana (window) está muy presente. Su filosofía es trabajar con colecciones/contenedores de datos referidos a un mismo concepto.

Es el contenedor principal del contenido que es visualizado en el navegador. A la hora de cargar una ventana, ya queda definido el objeto window en la memoria. El área de influencia de este objeto abarca toda la ventana y la barra de herramientas, inlcuidas la barra de desplazamiento y la barra de estado.

Los principales objetos que nos permitirán manipular y gestionar las diferentes carácterísticas del navegador: location, navigator, screen, history, document y window. Estos objetos tienen diferentes propiedades y métodos, los cuales nos permitirán acceder y modificar distintas partes de nuestra página HTML.

3.4.1. Acceso a propiedades y métodos

Para poder acceder a las propiedades y métodos del objeto debemos introducir el carácter "." seguido del nombre de propiedad o método. Debe continuar con paréntesis e incluir en su interior todos los parámetros.

Mediante la cláusula self, hacemos referencia al objeto window. Como este objeto siempre está activo, podemos omitir la referencia a él.

Desde JavaScript no es posible crear la ventana principal de un programa. Una vez ejecutado el script, se podrá crear o abrir nuevas subventanas. Debemos utilizar el método open que está dentro del contenedor window. Contiene hasta tres parámetros que definen las características de esta nueva ventana: la URL del documento que se abrirá, el nombre de la ventana y la apariencia física (color, tamaño, etcétera).

                
                    // Abrir una ventana con un tamaño determinado y
                    se añadirá el contenido del documento new.html
                    var subVent = window.open(“new.html”,”new”,”height
                    =800,width=600”);

La cláusula windows.open la hemos asignado a la variable subVent. Podremos referenciar a la nueva ventana desde el script original de la ventana principal a lo largo de nuestro código. Para cerrar utilizaremos el método close().

                
                    subVent.close();

3.7.- Generación de texto y elementos HTML desde código JavaScript

Uno de los principales objetivos, es convertir una página web estática en una aplicación dirámica. Un ejemplo es el reconocimiento del navegador que se está utilizando por parte de los scripts para así trabajar con las etiquetas adecuadas para ese navegador.

Las ventanas de un navegador presentan un documento HTML y se representan mediante un objeto window que contiene un subojeto document.

Permite manipular y acceder a los objetos que representan al contenido. Es posible crear documentos dinámicos en lugar de estáticos. Se puede realizar gracias al método document.write(), el cual nos premite escribir resultados por pantalla.

Es posible utilizar este generador de contenido de dos formas:

  • Mostrar resultados en la ventana actual del navegador a través de secuencias de intrucciónes.
                
                    < script type = “text/javascript”>
                        var SO = navigator.platform;
                        document.write(“< h1>Se ha abierto con el navegador:
                        ” + SO + ”< /h1>”);
                    < /script>
  • La otra posibilidad el método document.write()
                
                    < script type = “text/javascript”>
                        var texto = “Título nueva página”;
                        var ventana = window.open();
                        ventana.document.write(“< h1>” + texto + ””);
                    < /script>

3.9.- Cookies

Las cookies son una serie de archivos de texto que las aplicaciones almacenan en el ordenador del usuario y que guardan datos que la aplicación puede recuperar. El protocolo HTTP no tiene ningún estado, por lo que cada petición es independiente de la anterior.

Se envían de forma automática en la cabecera de cada petición de forma que permiten a las aplicaciones obtener peticoines anteriores gracias a los datos almacenados en ellas.

El tamaño máximo que puede almacenar una cookie es de 4KB de datos y se permiten hasta 20 cookies por dominio.

Lectura y grabación de las cookies

Para leer y grabar contenido en las cookies, haremos uso del objeto document. A esta propiedad se le asigna un texto que se almacena en la cookie.

                
                    document.cookie = “nombreCookie = valor”;
                
            
                
                    document.cookie = “usuario = ilerna”;

Cada nueva cookie puede utilizar este mismo formato, tener el mismo nombre y cambiar su valor.

Si se quiere modificar el valor de una cookie, se indica el mismo nombre y se cambia el valor.

                
                    document.cookie = “usuario = online”;

Se puede leer el contenido de una cookie visualizando el contenido de esa propiedad:

                
                    console.log(document.cookie);

Posible resultado:

                
                    nombre=online; lenguaje=es; fechaUltimaSesion=
                    15/09/2020;

Fecha de expiración

Por defecto, las cookies son eliminadas cuando un usuario finaliza la sesión. Este fin de sesión se produce cuando se cierra el navegador.

En ocasiones, se busca que las cookies sean persistentes, que los valores se mantengan entre sesión y sesión. Para conseguir esto, es necesario indicar que una fecha de expiración futura, es decir, indicar el tiempo máximo que vivirá esa cookie.

La forma en que se graba la fecha de expiración es obteniendo la fecha actual y posteriormente añadiendo el tiempo de duración de la cookie. Se consigue a través del objeto de tipo Date en milisegundos y posteriormente pasarlo a UTC.

                
                    //Crear objeto tipo fecha
                    let hoy = new Date();
                    //Crear variable obteniendo los milisegundos de
                    hoy
                    //Sumar los milisegundos correspondientes a una
                    semana
                    let caducidadMs = hoy.getTime() + 1000 * 60 * 60 *
                    24 * 7;
                    let caducidad = new Date(caducidadMs);
                    //Colocamos la fecha en formato UTC
                    document.cookie = “usuario=ilerna;expires=${caducidad.
                    toUTCString()}”;

Borrar cookies

Para borrar una cookie se debe indicar una fecha de expiración del pasado.

                
                    document.cookie = “usuario=ilerna;
                    expires=Sat, 01 Jan 2000 00:00:01 GMT”;

Al indicar una fecha anterior a la actual, la cookie queda caducada y el navegador automáticamente la eliminará.
