Tema 3. - Identificación y aplicación de los objetos predefinidos del lenguaje

Mediante el modelo de objetos del documento(DOM) es posible visualizar el documento de una forma diferente, ya que define el contenido de un determinado documento como un conjunto de objetos sobre los que JavaScript puede interactuar.

El modelo de objetos del documento(DOM) como una interfaz de programación de aplicaciones (API) para aquellos documentos HTML válidos y para los XML que estén bien construidos. Además, el DOM puede definir la estrucutra lógica de estos documentos junto con la forma para poder acceder a ellos y manipularlos.

Un objeto es una colección de propiedades que, para definir un estado, van a utilizar una serie de funciones o métodos que harán funcionar a esas propiedades.

Para la creación de un objeto se utiliza la palabra clave new.

                
                    var nombreobjeto = new Object();

Para poder acceder a las propiedades de un objeto, utilizaremos el punto(.) y el nombre de la propiedad a la cual queremos acceder:

                
                    nombreobjeto.propiedad

En cambio, para acceder a un método concreto de un objeto:

                
                    nombreobjeto.metodo([parámetros ocionales])

Cuando necesitaremos referenciar alguna propiedad por su índice, siempre comienza por 0.

3.1.- Utilización de objetos. Objetos nativos del lenguaje

Los objetos nativos del lenguaje JavaScript, aquellos de que disponemos para poder utilizarlos en diferentes aplicaciones. String, Math, Number y Date.

Permite la manipulación de cadenas de texto. Dispone de una propiedad que muestra el tamaño de la cadena. Posee diferentes métodos que nos permiten aplicar a toda variable a la que queramos asignar como valor una cadena de texto.

Permite una serie de operaciones matemáticas. No actúa como constructor, cada vez que deseemo llamar a alguna de sus propiedades o métodos, debemos hacerlo anteponiendo Math a dicha propiedad u objeto.

3.1.3.- El objeto Number

Poco utilizado, Number dispone de una información utilizada por programadores más específicos.

Las propiedades del objeto Number se refieren al rango de valores de números que soporta el propio lenguaje, siendo el más alto el 1.79E + 308 y el más bajo, 2.22E - 308.

Los números más altos que el número mayor se considera como infinito positivo, y los más pequeños que el menor, como infinito negativo.

Todos los números junto con sus correspondientes valores se definen como valores de doble precisión de 64 bits.

Entre las principales propiedades:

Propiedad Descripción
MAX_VALUE Devuelve el mayor número disponible en JavaScript
MIN_VALUE Devuelve el menor número disponible en JavaScript
NEGATIVE_INFINITY Representa el infinito negativo
POSITIVE_INFINITY Representa el infinito positivo
NaN Representa el valor especial Not a Number

Método

Método Descripción

3.1.4.- El objeto Date

Permite realizar operaciones con el tiempo en las aplicaciones web. Se utiliza para trabajar con fechas y horas.

Se van a crear mediante new Date() y lo podemos hacer de cuatro formas diferentes:

                
                    1. var d = new Date();
                    2. var d = new Date(milisegundos);
                    3. var d = new Date(cadena de Fecha);
                    4. var d = new Date(año, mes, día, horas, minutos,
                    segundos, milisegundos);

No presenta ninguna propiedad, tiene una serie de métodos que podemos dividirlos en tres subconjuntos:

  • Métodos de lectura: aquellos que empiezan con el prefijo get. La principal función de estos métodos es consultar las diferentes partes de una instancia del objeto Date.
  • Métodos de escritura: empiezan con el prefijo set. La utilidad de estos métodos es inicializar las diferentes partes de una instancia del objeto Date.
  • Métodos de conversión: convierten objetos de tipo Date en cadenas de texto o en milisegundos según los diferentes criterios que veremos a continuación.
Métodos del objeto Date
Método Descripción

3.3.- Clases

El lenguaje JavaScript está basado en prototipos y cada uno de sus objetos tiene una propiedad interna oculta nombrada [[Prototype]] que puede tener propiedades y métodos de objetos.

Las clases surgen como una mejora sobre la herencia basada en estos prototipos. No introducen un nuevo modelo, sino que aportan una sintaxis mucho más simple y clara para crear objetos y lidiar con la herencia.

Las clases son funciones especiales. La sintaxis de la clase tiene dos componentes:

  • Expresiones de clases
  • Declaraciones de clases

Para la declaración, utilizaremos la palabra reservada class junto a un nombre.

                
                    class Alumno {
                        constructor(nombre, apellido, edad) {
                            this.nombre = nombre;
                            this.apellido = apellido;
                            this.edad = edad;
                        }
                    }

Una diferencia entre declarar funciones y clases es que las declaraciones de funciones son izadas y las declaraciones de clases no lo son, tenemos que declarar la clase antes de realizar una llamada para crear objetos de esta.

                
                    var a = new Alumno (); // ReferenceError
                    class Alumno {}

También podemos definir las clases con una expresión de clase. Pueden ser nombradas o anónimas. El nombre dado a la expresión de clase nombrada es local dentro del cuerpo de esta.

                
                    // Anónima
                    var Alumno = class {
                        constructor(nombre, apellido, edad) {
                            this.nombre = nombre;
                            this.apellido = apellido;
                            this.edad = edad;
                        }
                    };
                    // Nombrada
                    var Alumno = class Alumno {
                        constructor(nombre, apellido, edad) {
                            this.nombre = nombre;
                            this.apellido = apellido;
                            this.edad = edad;
                        }
                    };

El cuerpo de una clase es la parte ubicada entre las llaves {}, donde se definen los miembros de clase, los métodos o contructores y todas las funcionalidades que queremos que realice la clase.

El método constructor es un método especial utilizado para crear e inicializar un objeto creado con una clase. Solamente puede haber un método especial con el nombre constructor en una clase. Si una clase contiene más de un método constructor, se lanzará un error SyntaxError.

Un constructor, para llamar al constructor de su superclase, utilizará la palabra super.

                
                    class Alumno {
                        constructor(nombre, apellido, edad) {
                            this.nombre = nombre;
                            this.apellido = apellido;
                            this.edad = edad;
                        }

                        get Nombre() {
                            return this.NombreCompleto();
                        }

                        NombreCompleto() {
                            return this.nombre + ‘ ‘ + this.apellido;
                        }
                    }
                    
                    function MostrarAlumno(){
                        const alumno = new Alumno(‘Ilerna’, ‘Online’,
                        23);
                        alert(‘El nombre completo del alumno es: ‘
                        + alumno.Nombre);
                    }

Es posible llamar a los métodos estáticos sin instanciar la clase. Para definir un método como estático, utilizaremos la palabra clave static. Los métodos estáticos se utilizan a menudo para crear funciones de utilidad en una aplicación.

                
                    class Calculo {
                        static multiplicacion(a, b) {
                            return a * b;
                        }
                    }
                    function MostrarMult(){
                        alert(Calculo.multiplicacion(5,10));
                    }

Para crear clases hijas de una clase genérica que tengamos (clase padre), utilizaremos la palabra clave extends.

                
                    class Persona {
                        constructor(nombre) {
                            this.nombre = nombre;
                        }

                        Situacion() {
                            return this.nombre + ‘ tiene relación
                            con Ilerna Online.’;
                        }
                    }

                    class Profesor extends Persona {
                        Situacion() {
                            return this.nombre + ‘ imparte clases
                            en Ilerna Online.’;
                        }
                    }

                    class Alumno extends Persona {
                        Situacion() {
                            return this.nombre + ‘ recibe clases
                            en Ilerna Online.’;
                        }
                    }

                    function mostrarPersona(){
                        var nombre = document.getElementById(‘nombrePersona’).value;
                        if(document.getElementById(‘profesor’).checked){
                            const profesor = new Profesor(nombre);
                            alert(profesor.Situacion());
                    }   else if(document.getElementById(‘alumno’).checked){
                            const alumno = new Alumno(nombre);
                            alert(alumno.Situacion());
                    }   else{
                            const persona = new Persona(nombre);
                            alert(persona.Situacion());
                        }
                    }

Las clases tradicionales basadas en funciones también podemos extenderlas a otras subclases.
