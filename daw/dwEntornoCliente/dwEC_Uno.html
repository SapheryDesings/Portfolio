Desarrollo web en Entorno de Cliente Principal

Tema 1. - Selección de arquitecturas y de herramientas de programación

Un progamador web debe dominar todas las fases por las que pasa un proyecto, captar las necesidades del cliente, definir unos objetivos y una estrategia, desarrollar una interfaz gráfica y elaborar prototipos y una presentación final. Es imprescindible conocer el orden de cada fase y qué realizar en cada una de ellas.

Es un campo multidisciplinar

1.1.- Modelos de ejecución de código

El diseño web comprende los siguientes campos:

  • Visual: su objetivo es la comunicación e intentar atraer a los clientes. Aumenta las posibilidades de que estos vuelvan a visitar.
  • Tecnología: de esta elección dependerá las características del sitio web, si será una página estática o dinámica, qué bases de datos se utilizarán.
  • Distribución: la velocidad y fiabilidad de la transmisión de datos.
  • Propósito: debe tener un objetivo, con una finalidad económica o informativa.

La arquitectura está basada en el modelo cliente-servidor.

  • El servidor, donde tenemos instalados los elementos hardware y software, así como las técnicas incrustadas y los lenguajes de programación instalados en la máquina. Aconsejamos un lenguaje de programación que soporte multihilos.
  • Se dispone de la máquina cliente, donde tenemos implementados los programas que realizan la petición de solicitud web al servidor a través de los navegadores, que deben ser capaces de soportar las tecnologías del sitio web, como HTML o CSS, así como otros lenguajes, como JavaScript y controles ActiveX.
  • La conectividad de la red y las características utilizadas para el sitio web.

El objetivo de un navegador es presentar la información, escrita normalmente en lenguaje HTML.

1.2.- Lenguajes de programación en entorno cliente

También surge la necesidad de implementar un lenguaje que interactúe con el usuario. Se consigue que la página web sea más atractiva.

Lenguajes utilizados en la programación web, los basados en entorno cliente, que son ejecutados por los clientes en un navegador web, y en entorno servidor.

Debido al gran abanico disponible de lenguajes orientados a la programación web, es de vital importancia saber elegir el lenguaje web que se implementará en una página. Cada lenguaje presenta sus ventajas e inconvenientes. Es habitual que la última palabra la tenga el desarrollador, dependiendo del lenguaje con que mejor se desenvuelva.

En las páginas web dinámicas los lenguajes que se utilizan en los dos entornos (cliente y servidor) se complementan.

Con un lenguaje orientado a entorno servidor necesita recoger las peticiones que se realizan desde un leguaje de entrorno cliente y tener la posibilidad de guardar la información en una base de datos.

Tendría que comprobar la información recogida en un formulario web antes de que esta sea almacenada en la base de datos del servidor. La validación de los datos se realiza mediante un lenguaje de cliente, mientras que el almacenamiento en la base de datos se realiza mediante un lenguaje de servidor.

Lenguaje en entorno cliente, para tener una página lo más completa posible se recomienda la combinación de varios lenguajes. Por este motivo, es posible diferenciar dos tipos de lenguajes:

  • Lengujes que permiten dar formato al texto y al estilo de la página (HTML y CSS).
  • Lenguajes que permiten crear páginas dinámicas con lenguajes de script (JavaScript).

Las cuatro capas de desarrollo web en el lado del cliente. JavaScript se sitúa en la capa superior, gestionando el comportamiento de la página web.

1.3.- Mecanismos de ejecución de código en un navegador web

Una serie de elementos y componentes determinados que conforman lo que se denomina arquitectura del navegador. La gran mayoría de ellos coinciden en una serie de componentes básicos y comunes, es lo que llamamos arquitectura de referencia.

  • Interfaz de usuario: capa que actúa como interfaz entre el usuario y el motor del buscador. Visualización de barras de herramientas, progreso de carga de la página, gestión inteligente de las descargas, preferencias de configuración de usuario o impresión. Puede mantener comunicación con el sistema operativo para el manejo de sesiones y preferencias de visualización o configuración.
  • Motor del buscador: es un componente que ofrece una interfaz para el motor de renderizado. Su función es cargar una dirección determinada (URL o URI) y soportar mecanismos básicos como volver a la página anterior o siguiente o recargar la página. Además, carga las alertas de JavaScript.
  • Motor de renderizado: se encarga de presentar de manera visual el recurso obtenido a partir del acceso a una dirección web. Interpretara el código de la página web. Capaz de mostrar documentos HTML, XML, CSS e incluso contenido embebido en la página e imágenes.
  • Subsistema de comunicaciones: implementar los protocolos de transferencia de archivos y los documentos que se utilizan en internet.
  • Intérprete de JavaScript: es el responsable de analizar y ejecutar dicho código, como pueden ser eventos del ratón o teclado u otro código intercalado para ciertas funcionalidades.
  • Parser XML: facilitar el acceso a los contenidos definidos en un documento HTML, este módulo permite cargar en memoria una representación en árbol (DOM) de la página, facilitando y agilizando así el acceso a los elementos de una página.
  • Componente de visualización: ofrece funcionalidades para la visualización del contenido de un documento HTML.
  • Subsistema de persistencia de datos: almacena diferentes tipos de datos para los principales subsistemas del navegador, como historiales de navegación, mentenimiento de sesiones de usuario o preferencias de configuración del navegador. A nivel más bajo, también almacenan certificados de seguridad y cookies.

1.4.- Capacidades y limitaciones de ejecución

Con el objetivo de mejorar la usabilidad del usuario:

  • JavaScript no puede escribir ficheros en el entorno servidor, una de sus funcionalidades es preparar los datos que van a ser enviados o ceder los datos para que estos sean tratados en otro lenguaje.
  • Se tendrá que preparar nuestro sitio web para que sea navegable sin la opción de JavaScript. Se utilizará para crear una navegación rápida, moderna y/o divertida, pero, en caso de que el navegador no tenga la opción, la web no deberá quedarse inactiva.
  • Los datos obtenidos desde el lado del servidor serán examinados mediante eeste lenguaje y actualizará las diferentes secciones o partes de la web.

JavaScript da soluciones a:

  • Dar un feedback al usuario desde formulario o hipertexto.
  • Ofrecer una interfaz amigable.
  • Controlar los plugins, marcos de navegación o applets de Java.
  • Procesar los datos del formulario antes de ser enviados.
  • Modificar la forma dinámica los estilos y contenidos en los navegadores.

1.6.- Integración del código con las etiquetas HTML

Los navegadores aceptan la integración dle código en HTML de varias formas. Las etiquetas script y el tributo type, que indicará el tipo de lenguaje que utilizaremos.

Otra forma es mediante un fichero externo que contenga el código. Es más recomendable, tendremos la opción de compartir el mismo fichero para varios archivos HTML y facilitar la depuración de los errores, así como contribuir a la claridad en el entendimiento del código, la modularidad, seguridad y rapidez a la hora de cargar la web.

A la etiqueta script se le añadirá el atributo src con el nombre del fichero, la extensión será .js.

Para cargar más de un fichero tendremos que repetir la etiqueta script pero cambiando el nombre del fichero fuente. Podemos utilizar la ruta relativa como la ruta remota.

Los navegadores que no soporte los scripts debemos informar al usuario. La marca que se deben utilizar para esta situación es noscrpt.

1.7.- Herramientas de programación sobre clientes web. Herramientas y lenguajes asociados

Una herramienta para desarrollar código, editor de texto.

  • No utilizar los editores orientados a modificación de contenidos
  • La facilidad para almacenar ficheros HTML, la codificación de colores, la verificación de la sintaxis, la inserción de comentarios, generación de bloques de códigos o la disposición de opciones extra, como clientes FTP.

Para la realización de un portal web:

  • Empezaremos codificando los códigos HTML, JavaScript y CSS en el documento.
  • Lo guardaremos con sus respectivas extensiones y lo ejecutaremos en los navegadores. Si se realizasen modificaciones, tendríamos que recargar el navegador.
  • La validación es el último paso, se ahorran horas de comprobaciones, aseguramos la validez del código y detectamos las imperfecciones que poseen los distintos archivos que componen el portal web.

La estandarización dle consorcio W3C es la que rige las validaciones, en su dirección web es donde podemos acceder para esta operación.
