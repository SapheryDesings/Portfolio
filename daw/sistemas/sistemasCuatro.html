Tema 4.- Administración de dominios

>Introducción

Mediante la Administración dominios podemos centralizar, simplificar y agilizar muy significativamente las tareas de gestión. Crear un dominio en un equipo servidor al que se conecten equipos cliente resulta muy práctico para administrar redes.

Estructura cliente-servidor

Este modelo de comunicación es bastante sencillo, ya que apartir de un equipo cliente, se necesita otro equipo servidor para que este realice aquellas funciones que tiene asignadas.

Pueden ser equipos diferentes o iguales. Una computadora puede ser cliente y servidor a la misma vez depediendo del software.

Consta de los elementos:

  • Cliente: es el sistema que precisa de servicios
  • Servidor: es el sistema que proporciona servicios

Cliente y servidor son elementos separados que establecen su comunicación mediante la red de comunicaciones y colaboran para completar tareas de forma conjunta. Cuando el cliente precisa un servicio, realiza una petición y el servidor le da una respuesta. El servidor recibirá y procesará la petición y le brindará el servicio requerido. Ejemplo, el servicio web que empleamos para navegar por internet.

Protocolo LDAP

Consideramos LDAP(Lightweight Directory Access Protocol) como un protocolo ligero de acceso a directorios. Se utiliza especialmente en las empresas, se emplea para acceder a servicios de directorio.

Podemos considerarlo como una base de datos pensada para almacenar directorios. La información se organiza de manera jerárquica y cada directorio puede almacenar un amplio abanico de datos.

Las características y funcionalidades que nos brinda son muchas. Las más destacables son:

  • Acreditación de usuarios:

    Los usuarios pueden acceder a sus respectivas cuentas desde cualquier equipo en red acreditado mediante LDAP.

  • Búsqueda de datos:

    También pueden emplearlo como si se tratara de una guía virtual que brinda los datos de contacto de otros usuarios de manera sencilla y accesible.

  • Centralización de la administración:

    Permite centralizar la gestión de cuentas de usuario y sus permisos asociados.

  • Posibilidad de replicar la base de datos:

    Una vez hayamos configurado la base de datos podemos replicarla. Lo que nos permite la opción de configurar diversos servidores LDAP sincronizados. Cuando la red y cantidad de usuarios crecen rápidamente, la red puede balancear la carga entre varios servidores.

LDAP brinda gran rapidez en lectura y escritura de datos, incluso cuando se da un gran volumen de accesos simultáneos. Cuando un cliente se conecta al servidor puede consultar o modificar directorios. En el segundo caso el servidor verifica los permisos necesarios antes de actualizar la información.

En la distro Ubuntu podemos instalar el paquete OpenLDAP y sus utilidades en el servidor tecleando:

“sudo apt-get install slapdldap-utils”

En Windows también podemos utilizar OpenLDAP.

El sistema de Microsoft se refiere a su implementación del servicio de directorio en red como Active Directory. Este emplea varios protocolos, entre los que se encuentra LDAP. En Windows Server se instala por defecto, en Windows on podemos instalar Active Directory pero si gestionarlo remotamente a traves de la red mediante la aplicación Sitios y servicios de Active Directory.

Concepto de dominio. Subdominios. Requisitos necesarios para montar un dominio

Dependiendo de si estamos hablando de redes en sentido general o bien de internet podemos encontrar dos deficiones diferentes.

  • Dominios de redes:

    De redes en general, comprende un determinado grupo de ordenadores que están conectados y que han otorgado a uno de los equipos la gestión de los usuarios, sus privilegios de acceso y otros datos.

  • Dominios en internet:

    Hace referencia a la parte principal de una determinada dirección web. Dicha parte hace referencia a la empresa u organización que gestiona el sitio.

El subdominio, también puede variar dependiendo de a qué nos refiramos:

  • Subdominios en redes:

    Son subgrupos o subclasificaciones dentro de un dominio. Su creación implica la creación de un dominio de segundo nivel, anidado dentro del dominio primario. La nueva estructura está relacionada con la primera, pero cuenta con características propias. Se utilizan con fines administrativos o para separar los diversos bloques de una organización.

  • Subdominios en internet:

    Suele hacer referencia a la parte de la dirección web que antecede al dominio y queda separado de este por un punto. Se emplea para compartimentar grandes sitios web en diferentes áreas. Con otros fines podemos usarlos para hacer pruebas y desarrollar nuevas páginas o aplicaciones. Si trabajamos en un subdominio, mantendremos intacta la estructura básica y nuestro posicionamiento no se verá afectado.

Requisitos para la creación de un dominio de red

El principal requisito es disponer de un S.O. orientado a servidor. Ejemplo Windows Server. Entre las tareas que realiza, destacar la de asignar CPU y memoria a los procesos.

Para configurar el dominio debemos instalar los roles necesarios para gestionar el dominio de red.

El rol DNS nos facilita esta tarea. Debemos instalar los roles de AD/DS y DHCP para complementar el funcionamiento del primer rol de dominio.

Una vez instalado los roles debemos configurar toda la infraestructura del dominio.

Configuramos el servicio DNS, estableciendo un servidor principal de dominio y un usuario administrador. Para finalizar debemos promover el servidor a controlador de dominio, donde estableceremos características como el nombre que sea concreto y orientativo.

Debemos generar las zonas de búsqueda directa e inversa en el servidor DNS.

Establecer en el servicio DHCP el rango de IP a proporcionar a los equipos clientes conectados con el servicio DHCP.

En estas configuraciones podemos realizar todas las configuraciones del dominio, como las reservas de direcciones, configuraciones de correo, etc.

En Linux podemos utilizar Ubuntu Server. Para crear el dominio tenemos que configurar el servidor DNS: “sudo apt-get install -y bind9 ”.

Entramos en el directorio /etc/bind/ para editar los archivos de configuración.

Las configuraciones se realizaran mediante la modificación de los ficheros “named.conf.options” donde configuraremos las opciones de bind y la comunicación entre la red privada.

En el fichero “named.conf.local” junto con el fichero “zones. rfc1918” configuraremos las zonas de búsqueda.

En el directorio “var/cache/bind” los ficheros para configurar las direcciones de los equipos conectados al dominio.

Requisitos para la creación de un dominio en internet

Debemos contratar el servicio con una empresa. La administración se lleva a cabo de manera remota, a través del navegador web.

Conceptos clave de Active Directory

Es una herramienta de Microsoft para gestionar servidores y administrar en ellos usuarios, grupos, etc.. Podemos gestionar los inicios de sesión de los equipos conectados a la red, definir políticas, etc.

Debemos aprender la terminología que emplea:

  • Objeto:

    Genérico para referirnos al cualquiera de los componentes que conforman el directorio. Cada objeto puede tener sus características y un nombre para identificarlo.

  • Directorio:

    AD se basa en el concepto de directorio, que es un repositorio en el que se guarda toda la información referente a usuarios, grupos, recursos, etc.

  • Dominio:

    Conjunto de objetos dentro del directorio que conforman un subconjunto administrativo. Dentro de un "bosque" puede haber varios dominios. Cada uno puede tener su propio conjunto de objetos y unidades organizativas.

  • Controlador de dominio:

    Conjunto de objetos del directorio para un determinado dominio. En un dominio puede haber varios controladores de dominio.

  • Árboles:

    Son conjuntos de dominios que poseen una raíz común. Están organizados jerárquicamente y su jerarquía se refleja en los nombres.

  • Bosque:

    Abarca todos los dominios dentro de su ámbito, por otra parte están interconectados por lo que se denomina Relaciones de confianza. Todos los dominios de un bosque confían automáticamente entre si, los distintos árboles pueden compartir recursos entre sí. Un bosque contiene siempre al menos un dominio, que ejerce de raíz del bosque.

  • Unidades organizativas:

    Contenedores de objetos que permiten organizarlos jerárquicamente en subgrupos dentro del dominio. Podemos definir estructuras lógicas que faciliten la organización y hagan que la administración sea más sencilla.

  • Relaciones de confianza:

    Son un método de comunicación entre dominios, árboles y bosques que rige la seguridad de la red. Los usuarios de AD pueden autenticarse en otro dominio del directorio.

    Pueden ser de diversos tipos, las más comunes:

    • Unidireccionales: funcionan en una única dirección
    • Bidireccionales: funcionan en ambas direcciones
    • Transitivas: en ellas la confianza se propaga
  • Delegación de control entre dominios:

    Permite a los usuarios de un dominio administrar recursos de otro dominio. Es necesario entre los dominio una relación de confianza. La delegación de control la realizaremos solamente a usuarios en los que confiemos plenamente.

En AD podemos establecer distintos tipos de grupos. Se emplean para reunir a usuarios, equipos y otros tipos de cuenta en entidades administrables.

Administración de cuentas. Cuentas predeterminadas

En aplicaciones como AD nos interesa generar cuentas de usuario para asignarlas a las distintas personas que van acceder a la red. A nivel interno, las cuentas de usuario son elementos de una base de datos de directorio a los que se les asigna de forma automatizada unos identificadores para garantizar la seguridad. Tras dar de alta a los usuarios, estos pueden acceder y utilizar los recursos del dominio que tengan asignados.

En Windows podemos gestionar las cuentas entrando en Server Manager y posteriormente en Local Server. Agregar y eliminar usuarios se llevan a cabo desde AD User and Computers.

Nos permitirá gestionar los usuarios y grupos del dominio de una forma muy similar a las versiones cliente de Windows. También podmos gestionar los equipos enlazados al dominio u otros servidores que gestionen la red, con el objetivo de tener un mayor control sobre las conexiones en dominio.

Los servidores Linux son utilizados en modo texto. Los comandos más básicos para agregar o eliminar usuarios:

  • sudo adduser usuario: crea una cuenta de usuario
  • sudo deluser usuario: elimina una cuenta de usuario

En Linux podemos instalar diferentes interfaces gráficas y cada una pueden tener unas opciones diferentes de gestión.

En el listado de usuarios, encontramos tres cuentas predeterminadas:

  • Administrador (Plenos derechos)
  • Invitado (Permisos muy limitados)
  • Asistente de ayuda

Cada una tiene asignada una combinación distinta de privilegios y permisos.

La instalación de los servicios de red como los dominio crea grupos de usuarios por defecto y se les asigna un usuario administrador del dominio, el cual tendrá plenos privilegios.

Las cuentas de usuarios del dominio suelen estar vinculadas a ciertos equipos para mejorar el rendimiento de la red, aunque con ellas podemos acceder a cualquier equipo que esté conectado al dominio, teniendo siempre los permisos necesarios.

Contraseñas. Bloqueos de cuenta. Cuentas de usuarios y equipos

A los usuarios dados de alta se les asignarán distintos privilegios, por lo que es vital que le asignemos contraseñas seguras.

Servicios como AD se encargan de autentificar la identidad de los usuarios. Cuando una persona inicia sesión en el dominio, se comprueban sus credenciales en el directorio. Deben tener su propio nombre de usuario y contraseña.

Tras solicitar el acceso a los recursos mediante la introducción de los datos, pueden tener acceso si son correctos, en caso contrario se les deniega el acceso.

Para impedir que los usuarios no autorizados realicen infinitos intentos de entrar, podemos definir unos parámetros que la bloquearán.

Mediante otros parámetros, como Duración de bloqueos o Restablecer la cuenta, podemos restaurar la cuenta pasado un intervalo de tiempo.

Otros bloqueos pueden ir en función de si mantienen su sesión iniciada durante demasiado tiempo, etc...

En AD > Configuración de seguridad local > Directiva de bloqueo de cuentas.

Perfiles móviles y obligatorios

Los perfiles de usuario nos permiten asignar a los usuarios del sistema características como los archivos y carpetas que utilizarán y la configuración de su escritorio. Toda esa información se lee y se carga cada vez que un usuario inicia sesión.

Por defecto, los perfiles suelen ser locales. Tenemos la posibilidad de crear un perfil móvil. Se trata de que el usuario tenga siempre acceso a su perfil, independientemente del equipo desde el que inicie sesión.

Este perfil lo puede realizar el administrador o un usuario con privilegios. Lo definirán en un directorio del servidor, de modo que quede compartido en red.

Los perfiles obligatorios se definen por el hecho de ser de solo lectura y únicamente pueden ser modificados por un administrador. Cuando el ususario inicie sesión se cargará su perfil pero no se guardará ninguna configuración o dato cuando cierre sesión.

En caso de un perfil móvil obligatorio, tenemos que introducir la información correspondiente a la ruta del acceso en el perfil, le agregamos la extensión .man.

Para convertir un perfil móvil en uno obligatorio accedemos al archivo ntuser.dat, y le cambiamos la extensión por ntuser.man.

Carpetas personales

Cada usuario puede almacenar su información personal en una subcarpeta propia. Esta tendra el nombre del propio usuario y es frecuente que aparezca en los equipos cliente como una unidad de red.

Hay que tener en cuenta que los directorios que indiquemos para los usuarios del dominio tiene que ser recursos compartidos del servidor.

Estos serán visibles para los usuarios que indiquemos, aunque esots se podrían asignar directamente a grupos de usuarios para agilizar el proceso. Hay que definir muy bien los permisos que damos a cada usuario para acceder a los recursos compartidos, ya que por defecto vienen con control total.

Plantillas de usuario. Variables de entorno

Se utilizan para ahorrarnos una gran cantidad de trabajo. Nos permiten definir una serie de características comunes que posteriormente podemos aplicar a un gran número de usuarios.

Las variables de entorno, conforman un conjunto de valores que se ejecutan al inicio de la sesión del usuario y afectan al comportamiento de comandos y procesos.

Las más comunes de Windows son:

  • %APPDATA%: directorio de las carpetas ocultas para datos de las aplicaciones
  • %USERPORFILE%: directorio del usuario de la sesión
  • %WINDIR%: ruta por defecto de la instalación de Windows
  • %PROGRAMFILES%: directorio que almacenan la instalación de los programas

Estas no siempre conducen a directorios. %TIME% nos devuelve la hora actual del sistema.

Como usuarios del sistema podemos editar y crear variables de este tipo. Programas como Oracle Database crean variables de este tipo para gestionar mejor sus recursos.

En caso de linux encontramos variables como:

  • 'SHELL': intérprete de comandos
  • 'LANG': idioma actual del sistema
  • 'HOME': directorio actual del usuario de la sesión

Para modificar los valores de estas variables debemos acceder a tres archivos de texto almacenados en el sistema

  • /etc/environment -> variables independientes del intérprete de comandos
  • /etc/profile -> para definir variables que serán válidas para todas las shells interactivas que exijan login. Si queremos definir únicamente variables de usuario el directorio es /.bash_profile
  • /etc/bash.bashrc -> igual que el anterior aunque para consolas no logueadas. El equivalente para variables de usuario es /.bashrc

Administración de grupos. Tipo. Estrategias de anidamiento. Grupos predeterminados

Si la empresa u organización cuya red administramos tiene un elevado número de departamentos y usuarios es fundamental que planeemos una estrategia para estructurarla en grupos.

Definir permisos y el acceso a los recursos resultará mucho más rápido. Los grupos nos permiten aunar las cuentas de usuario en unidades fácilmente administrables. Cuando añadimos un usuario a un grupo, este recibe todos los privilegios de usuario asignados al grupo y los permisos referentes a sus recursos compartidos.

Distinguimos entre los siguientes tipos de grupos:

  • Grupos de distribución: pensados solamente para usarse con aplicaciones de correo electrónico
  • Grupos de seguridad: se utilizan para administrar permisos de acceso a los diversos recursos de la red

Otra clsificación en función de su ámbito:

  • Grupos de ámbito universal: pueden emplearse en cualquier parte de un mismo bosque. Tras definir un grupo universal podemos asignarle usuarios, anidados y usar listas de control de acceso para definir permisos
  • Grupos de ámbito global: los permisos que conferiremos a este grupo son válidos en cualquier dominio, pero sus miembros solo pueden realizar acciones en el dominio en el que está dado de alta el grupo global
  • Grupos de ámbito local: los miembros dados de alta en este pueden realizar acciones en cualquier dominio, pero sus permisos son efectivos únicamente para recursos del dominio en el que fue creado el grupo.

Hay una serie de grupos que se crean de forma predeterminada:

  • Administradores
  • Operadores de copia de seguridad
  • Operadores criptográficos
  • Usuarios de COM distribuido
  • Invitados
  • IIS_IUSRS
  • Operadores de configuración de red
  • Usuarios del registro de rendimiento
  • Usuarios del monitor de sistema
  • Usuarios avanzados
  • Usuarios de escritorio remoto
  • Replicador
  • Usuarios
  • Ofrecer aplicaciones auxiliares de asistencia remota

En Linux al igual que sucedía con las cuentas de usuario tenemos dos formas de gestionarlo por comandos o interfaz gráfica.

Por comandos:

  • sudo addgroupnombreGrupo
  • sudo delgroupnombreGrupo

Es importante que tengamos presente que los grupos pueden anidarse entre sí. El anidamiento de grupos consiste en hacer un grupo miembro de otro, de tal modo que utilice sus características.

Instalación de Windows Server y Ubuntu Server

Empecemos por Windows Server 2019:

  1. Configuramos la BIOS para que se inicie desde el DVD
  2. Insertamos el disco de instalación de Windows Server 2019 y reiniciamos el PC
  3. Al iniciar, seleccionamos el idioma de la instalación
  4. Elegimos Instalar
  5. Elegimos la edición de Windows 2019
  6. Aceptamos la licencia del contrato
  7. Llegado este momento, podemos elegir entre Modo Recomendado o Modo Avanzado. Elegimos Modo Avanzado
  8. A continuación, llega el momento de elegir la partición del disco donde instalar el sistema operativo. Si queremos hacer la partición, entonces elegimos Nuevo, Tamaño de la partición y Formatear
  9. A partir de este paso, se instalará el sistema operativo y por último tendremos que reiniciar
  10. Escribimos la contraseña para el usuario administrador. Una vez iniciada la sesión, ya podemos empezar con la configuración básica de la administración del servidor

En el caso de Ubuntu Server 16, los pasos serán los que detallamos a continuación:

  1. Configuramos la BIOS para que se inicie desde el DVD
  2. Insertamos el disco de instalación y reiniciamos el PC
  3. Al iniciar, seleccionamos el idioma, país y la zona horaria
  4. Elegimos el Hostname
  5. Creamos la cuenta para usuario y su contraseña
  6. Elegimos el cifrado de la carpeta
  7. Ahora llega el momento de elegir el particionado del disco
  8. Le damos a Instalar
  9. Elegimos la configuración de las instalaciones
  10. Configuramos el gestor de arranque
  11. Hecho esto, GRUB se instalará en el sector de arranque de la partición
  12. Tras reiniciar el PC y entrar por primera vez con la cuenta administrativa, actualizamos el sistema tecleando lo siguiente:
    • “sudo apt-get update”
    • “sudo apt-get upgrade”
