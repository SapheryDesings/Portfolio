  • Sistema Informático: Es aquel que almacena y procesa información interrelacionando el hardware y software de un ordenador.

Para cubrir las necesidades que se nos plantean en el ámbito de la informática es preciso conocer en profundidad:

  1. Hardware: Antes de escoger el software debemos tener encuenta en que equipos vamos a instalarlo. Características técnicas, conectividad y prestaciones. Si los equipos están en red debemos tener muy clara la estructura para alcanzar el máximo rendimiento.
  2. Software: Debemos conocer el tipo de programa que se necesita, sus funcionalidades y la liciencia bajo la que se distribuye.

Estructura y componentes de un sistema informático, periféricos y adaptadores para la conexión de dispositivos

El sistema informático se compone de una parte central (ordenador) y unos dispositivos que facilitan la entrada/salida de informácion (periféricos). Un ordenador está formado por dos tipos de elementos:

  • Software: Componentes lógicos o programas.
  • Hardware: Componentes físicos y electrónicos.

Hardware

Podemos encontrar diferentes tipos, dependiendo del rendimiento; comentaremos los fundamentales:

  1. Caja, torre o chasis: Recinto donde se alojan los componentes. Se encarga de su protección, suele ser metálico o plástico.
  2. Fuente de alimentación: Es un transformador de corriente alterna a corriente continua con un voltaje apropiado para el funcionamiento de los componentes.
  3. Placa madre o placa base: Elemento principal, en el cual se conectan todos los componentes, también conocido como componente integrador.
  4. CPU (Unidad central de proceso): Se encarga de interpretar y procesar las intrucciones, a través de operaciones aritméticas, lógicas y entrada/salida del sistema. "El cerebro".
  5. Disco Duro: Es la memoria del ordenador, almacena archivos y programas, ya que es capaz de amacenar datos estando el odenador apagado.
  6. Memoria Ram (Random access memory): Permite intercambiar información entre el sistema operativo, el software, el procesador y otros dispositivos, tiene una gran velocidad de lectura/escritura, es un tipo de memoria volátil, no se guardan los datos cuando el equipo no tiene energía.
  7. Tarjeta gráfica: Se encarga de procesar los datos que son enviados desde el ordenador y transformarlos en datos visibles y comprensibles para el usuario.
  8. Tarjeta de sonido: Es una expansión que permite enviar información de audio a un dispositivo de audio.
  9. Tarjeta de red: Permite acceder a una red y compartir sus recursos.
  10. Otros: Unidades de almacenamiento adicionales. Periféricos de entrada/salida.

En windows, el administrador de dispositivos nos muestra una lista de todos los componentes conectados a nuestro ordenador. Podemos actualizar o instalar drivers para configurar determinados periféricos.

Software

  • Sistema Operativo
  • Aplicaciones
  1. Unidad central de proceso(CPU): Tiene como misión ejecutar las instrucciones de un programa. Está formada por circuitos que se encuentran integrados en un chip llamado microprocesador. Está compuesto por la unidad de control, la ALU, y los buses de entrada/salida.
  2. Unidad de control(UC): Es la parte pensante, su tarea es recibir información para interpretarla y procesarla mediante las órdenes que envía al resto de componentes.
  3. Unidad Aritmético-lógica(ALU): Calcula operaciones aritméticas y operaciones lógicas.
  4. Memoria principal: Es la memoria RAM. Es necesaria para poder procesar la información. Para que un programa pueda ser procesado, primero tiene que almacenarse en la memoria principal.
  5. Buses: Es una via de comunicación que conecta dos o más componentes. Funciones:
    • Soportar la información a transmitir
    • Garantizar la correcta comunicación entre los elementos
  6. Ciclo de instrucción: Es el conjunto de pasos que se realizan al procesar una instrucción de un programa. Está formado por la fase de búsqueda y la fase de ejecución.

El sistema informático como equipo único o conjunto de equipos de red, la conectividad es un término clave. Referencia la capacidad de los equipos y dispositivos para conectarse entre sí.

Todo sistema informático estará compuesto de hardware y software. En un sentido más amplio debemos añadir a los usuarios (Recursos humanos) y por supuesto, ls información.

Tipos y topologias de red

Una red de ordenadores es un conjunto de equipos interconectados entre sí que permiten compartir recursos e información. Es necesario que dispongan de tarjeta de red, cables de conexión, dispositivos perifericos y un software adecuado.

Tipos de red según su extensión

  1. PAN (Red de área personal): Cuando los equipos están situados a pocos metros y se utilizan solo para uso personal. Aparecieron para que el usuario no tuviera un solo equipo (ordenador) sino también tabletas, moviles y televisores. Para sacar provecho a estos dispositivos y las prestaciones que ofrecen es conveniente realizar una red entre todos.
  2. WLAN: Red lan inalámbrica, transmite información de forma inalámbrica.
  3. LAN(Red de área local): Dispone de un concentrador. A él se conectan todos los nodos de un edificio mediante un router.
  4. MAN(Red de área metropolitana): Puede abarcar desde varios edificios hasta una ciudad entera. Es la interconexión de varias LAN.
  5. WAN(Red de área extensa): Permite interconectar varias ciudades o países. Presentan una menor velocidad de transmisión y su tasa de error es mayor, se debe a la gran extensión que abarcan.

Toda red presenta una toplogía diferente, un diseño específico en los planos físicos y lógicos. Es en esencia un conjunto de equipos(nodos) interconectados, la manera en la que se establezcan las conexiones definirá su topología.

Topologías Básicas

  1. Punto a punto: Un canal para comunicar solamente dos nodos.
  2. Bus: Único canal de comunicaciones al que se conectan los distintos dispositivos.
  3. Anillo: Los dispositivos conforman un círculo de manera que cada equipo está conectado al siguiente, exceptuando el último círculo, que está conectado al primero.
  4. Doble anillo: Funciona igual que la topología en anillo. Además , posee un segundo anillo que conecta también todos los nodos.
  5. Arbol: Se estructura como un árbol, en el que los dispositivos o estaciones ejercen de ramas.
  6. Malla: Todos los nodos están conectados entre ellos. Para ir de un lugar a otro, la información puede seguir distintos caminos.
  7. Totalmente conexa: Igual que malla, aunque contiene más conexiones, ya que todos los nodos están conectados entre sí.
  8. Estrella: Los dispositivos están conectados siempre a un punto central y todas las comunicaciones se realizan a través de este nodo.
  9. Híbrida: Combina varias de las anteriores.

El modelo OSI describe como funciona una red y su estructura. Se trata de un modelo que se estructura en as, que definen las diferentes etapas por las que deben circular los datos para pasar de un dispositivo a otro de la misma red. Las capas son siete y se agrupan de la siguiente forma:

  1. Aplicación: La más cercana al usuario.
  2. Presentación: Se encarga de que la información enviada a la capa de aplicación pueda ser leída con éxito.
  3. Sesión: Garantiza la comunicación entre dos puertos (Hosts).
  4. Transporte: Transporta los datos del emisor hasta el receptor.
  5. Red: Responsable de que exista conectividad entre dos hosts diferentes.
  6. Vínculo de datos: Acceso al medio.
  7. Física: Transmite las señales.

Conectores de red

  1. BNC: Emplea cable coaxial y es más compleja de configurar. Actualmente se encuentra en desuso.
  2. RJ-45: Se emplean cables ethernet de 8 pines.
  3. Red inalámbrica: Ha supuesto una revolución en este ámbito, permite habilitar redes sin necesidad de cablearlas físicamente.

Mapa físico y lógico de una red

A la hora de crear o modificar una red es muy importante documentar su estructura. Deberá llevarse a cabo en dos frentes.

Mapa físico

Es una representación gráfica de una red en el mundo real. Todo deberá tenerse en cuenta para garantizar que la señal llegue a cada equipo de la mejor forma posible. Se realizan por aplicaciones que nos ofrecen subir los mapas como archivos en formatos para mostrar la representación y poder interacturar con los diferentes componentes o cosultar información de esta.

Mapa lógico

Documentación que hace referencia a los aspectos internos de la red. Es necesario saber que dirección ip tiene asignada cada ordenador, a que subred pertenece etc... Es una representación más técnica, que facilita el mantenimiento de la red, y la incorporación de nuevos nodos. Hay aplicaciones que nos ayudan a obtener la información necesaria de lso equipos, pero cuando se comprueba de forma masiva en redes muy grandes, se suele utilizar la consola de comandos.

Arquitectura de un Sistema Operativo

Los sitemas operativos han evoluciodo de forma notable, las consolas tipo texto han dejado paso a interfaces de tipo gráfico como windows o mac os x.

Podemos distinguir en la arquitectura los siguientes elementos clave:

  • Núcleo del Sistema: Componente que está ejecutandose permanentemente en la memoria. Se encargará de gestionar los recusrsos del ordenador, destacar la de asignar CPU y memoria a los procesos.
  • La API del núcleo: API(interfaz de programación de aplicaciones) comprende el conjunto de servicios que ofrece el sistema operativo a las aplicaciones estos pueden llamar al sistema operativo para valerse de él.
    Mediante dichas llamadas el sistema puede efectuar operaciones básicas como abrir archivos, modificarlos, cerrarlos, etc. Puede ejecutar instruciones de entrada/salida para operaciones relacionadas con gráficos y sonido, comunicaciones, etc.
  • El sistema de archivos: Con él obtenemos una estructura lógica de la información grabada en las unidades de disco y podemos trabajar con directorios y archivos. Es frecuente que forme parte del kernel o núcleo.
  • Controladores o drivers: Permite que el sistema interactue con los dispositivos hardware del equipo.
    La gestión se lleva a cabo a partir de ordenes que recibe el sistema operativo. Para introducirlas podemos usar la interfaz gráfica o la consola.

Funciones del Sistema Operativo

  • Administración de procesos: Decide el orden de procesamientos de los programas que están a la espera, así como asignar los recursos necesarios para su proceso.
  • Administración de recursos: Este proceso lleva un registro que le permite conocer qué recursos están disponibles y cuáles están siendo utilizados, por cuánto tiempo y por quién.
  • Control de operaciones entrada/salida: Decide que proceso hará uso del recurso, durante cuánto tiempo y en qué momento.
  • Administración de la memoria: Supervisa que áreas de memoria están en uso y cuáles están libres. Determina cuánta memeoria asignará a un proceso y en qué momento. Libera la memoria cuando ya no es requerida.
  • Recuperación de errores: El sistema operativo contiene rutinas que intentan evitar perder el control de una tarea cuando contiene errores en la transferencia de información hacia y desde los dispositivos de entrada/salida.
    1. Gestión y asignación de la memoria y la CPU.
    2. Gestión de las unidades de almacenamiento de la información.
    3. Gestión de las operaciones de entrada/salida.
    4. Mediación entre el hardware y el software a través de los drivers.
  • Gestión de usuarios y permisos: Administra los permisos a los usuarios que les permiten acceder al sistema y así evitar acciones que afecten al trabajo de otro usuario.
  • Control de seguridad: Para tener en cuenta la seguridad del software, usuarios y la información almacenada en el sistema.

Tipos de sistemas operativos

Entendemos por sistema operativo el software encargado de poner en marcha un ordenador, garantizando su correcto funcionamiento. Es el intermediario entre usuario y hardware. Los podemos clasificar:

Según la cantidad de procesos que pueden gestionar de forma simultánea

  • Monotarea: Un proceso en ejecución realiza tareas simultaneamente.
  • Multitarea: Ejecuta varios procesos a la vez, en dos procesos pueden utilizar dos CPU diferentes.

Según el número de usuarios

  • Monousuario: Único usuario trabaja con un solo ordenador.
  • Multiusuario: Varios usuarios pueden trabajar simultánemente.

Según el número de procesadores

  • Monoproceso: Solo un proceso a la vez. Realiza tareas rotatorias con intercambio muy rápido.
  • Multiproceso: Diferentes tareas al mismo tiempo, ofrece varios procesos de forma simultánea.

Según los servicios que ofrece

  • Centralizados: Todos los recursos se encuentran en una misma máquina.
  • Distribuidos: Los recursos pueden estar en diferentes máquinas conectadas en red.

Según el tipo de licencia

  • Propietarios: Limitaciones de uso. Propiedad de alguna empresa.
  • Libres: Permiten la modificación del producto. Usar el programa libremente.

Tipos de aplicaciones

  • Aplicaciones locales: Se almacenan en la unidad de disco local y solamente este tiene acceso a ellas.
  • Aplicaciones en red: Se ejecutan en un entorno de red local. Las aplicaciones primero se ejecutan de manera local y segundo de manera remota.
  • Aplicaciones web o en la nube: La mayor parte se ejecuta en un servidor remoto y queda accesible a través de internet. A veces los datos también se almacenan online.

Licencias y tipos

Licencias

  • Gratuitas: Permite la redistribución, no permiten modificar su código fuente.
  • Libres: Disponible para cualquier usuario, se puede utilizar, copiar y distribuir, en formato original o habiendo realizado modificaciones.
  • Comerciales: Desarrolladas por una empresa, su objetivo es obtener beneficio con su utilización.
  • Propietarios: Aquel cuyas copias, redistribuciones o modificaciones, están prohibidas por el propietario. Es necesario solicitar permiso a su propietario o pagar por ello.

El software comercial puede ser:

  • OEM: Forma parte de un equipo nuevo. Una vez activamos el hardware queda unido a esta licencia. Los fabricantes ponen limitaciones a su reinstalación. Está prohibida su venta y cesión a terceros, salvo si está especificado.
  • Retail: Suele tener un coste más elevado. Es posible reinstalarlo, cederlo a terceros o venderlo, ya que pasa a ser propiedad del usuario.
  • Por volumen: Destinado a grandes usuarios (Empresas). Similar a OEM, pero sin afectar a nuevos equipos. Determina el número de equipos en los que se utilizará. El fabricante está autorizado para comprobarlo.

Gestores de arranque

Los gestores de arranque son un pequeño programa que se ejecuta una vez iniciada la bios. Prepara los elementos más básicos para que el S.O. funcione.

Maquinas virtuales

Son un software que nos permite instalar S.O. de forma que crea que está instalado en un equipo. Hay dos tipos diferentes, las de sistemas y las de proceso. Emulan un ordenador por completo y ejecutan un S.O. en su interior.
Utilizan el hardware instalado en el equipo base, todo funcionará como si de un pc normal se tratase.

Las de proceso no emulan todo el equipo completo sino que ejecutan un proceso concreto como una aplicación. Son muy útiles para crear aplicaciones para varias plataformas, creando un entorno de ejecución para lidiar con el S.O. y no programar específicamente para un sistema.

Consideraciones previas a la instalación de sistemas operativos libres y propietarios

  • Requisitos del hardware: La documentación de los S.O. ofrece características mínimas requeridas.
  • Medios para la instalación del S.O.: Periféricos para leer los ficheros de instalación.
  • Undidad de destino: La partición o disco óptimo será la que disponga de mayor rápidez en la lectura/escritura de datos. Es importante planificar que el S.O. y los archivos se encuentren en diferentes particiones. La instalación del S.O. conllevará la destrucción de los datos de la partición elegida, antes será vital realizar una copia de seguridad.

Instalación de sistemas o requisitos, versiones y licencias

Los requisitos principales son:

  • Tipo y velocidad de la CPU
  • Cantidad de memoria RAM instalada
  • Espacio requerido en el disco duro
  • Tipo de tarjeta gráfica
  • Dispositivos necesarios (Lector DVD, puerto USB, etc..)

En la documentación también suelen especificarse unos requerimientos recomendados, es ideal que los cumplamos para obtener un rendimiento óptimo. Sistemas como windows ofrecen distintas modalidades (starter, home, ultimate, pro, etc..)
Si es de régimen propietario debemos adquirir una licencia.

Las particiones son divisiones lógicas en el disco físico. El S.O. trata cada partición como si fuera un disco independiente, de modo que podamos tener diferentes S.O. en particiones distintas.

Tipos de particiones

  • Primarias: Pueden ser reconocidas como particiones de arranque. Un disco cuenta con una única partición primaria que abarca todo el espacio disponible, está limitado a cuatro.
  • Extendidas o secundarias: Complemento o ampliación de las primarias que permiten definir más unidades lógicas. Se crearon para saltarse la limitación de cuatro particiones primarias por disco.

Instalación y desintalación de aplicaciones. Requisitos, versiones y licencias.

Antes de instalarlas debemos estar seguros de que cumplimos lo especificado en la documentación.

  • Requisitos: Nuestro hardware y sitemas operativos son aptos para la ejecución de las aplicaciones.
  • Versiones: La versión más reciente no será necesariamente la mejor. Debemos elegir la mejor para los fines que perseguimos.
  • Licencia: Si es shareware o comercial debemos actuar de acuerdo a la legalidad vigente.

La instalación implica llevar a cabo las operaciones precisas para que se pueda ejecutar. Normalmente disponen de un asistente, se distribuyen en un paquete comprimido que bastará con descomprimir.

La desinstalación es el proceso inverso y el S.O. brinda herramientas para ello.

Debemos señalar que las sucesivas instalaciones y desinstalaciones van dejando restos en los ficheros de configuración. Estos restos pueden incidir negativamente en el rendimiento.

Actualización de sistemas operativos y aplicaciones

  • Actualización: Instalar la nueva versión sin desinstalar la previa.
  • Instalación en limpio: A veces debemos desinstalar la versión previa antes de instalar la nueva.

En ocasiones su propia interfaz nos advertirá de actualizaciones disponibles. Los S.O. también advierten de actualizaciones disponibles, estas mejoraran y perfilarán funcionalidades y también redundarán en una mayor seguridad. En S.O. linux se pueden utilizar comandos o aplicaciones para comprobar las actualizaciones.

Archivos de inicio de S.O.

Los pasos para iniciar un ordenador varian dependiendo a diversos factores, dispositivos de arranque, existencia de particiones, etc...

  1. La bios inicia la pantalla y el teclado, verifica la memoria RAM, la fecha y otros datos. El orden de arranque de los periféricos es clave para que el sistema inicie correctamente.
  2. Luego, carga el gestor de arranque y seguidamente el S.O. Aquí se ejecutan los archivos de inicio. Se ocupan de cargar en memoria los servicios o los programas residentes.

Estos archivos pueden editarse con un editor de texto simple. En algunos casos contamos con software específico para operar cambios.

Registros del sistema

En determinados S.O. nos brinda un registro que centraliza la configuración, sus servicios y aplicaciones. El visor de eventos complementan la información del registro. Es preciso realizar mantenimiento y limpieza regular.

Actualización y mantenimiento de controladores de dispositivos

Drivers o controladores permiten que el S.O. pueden trabajar con ellos, conforme vayan apareciendo nuevas versiones de ellos para optimizar el rendimiento y minimizar problemas en el futuro.

  • Actualización desde el sistema: El S.O. permite comprobar si hay nuevas versiones y actualizarlas directamente.
  • Descarga online: Casi todos los fabricantes disponen de una web de donde descargar.

Dada su importancia lo ideal es respaldarlos en un momento en el que todo funcione correctamente.
