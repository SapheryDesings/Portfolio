Number

Declaración

Los números son uno de los tipos primitivos que para crearlos basta con escribirlos. También se pueden representar con objetos declarandolos mediante la palabra clave new, aunque no se suele utilizar ya que es bastante más complicado.

Constructor Descripción
new Number(numbre) Crea un objeto numérico a partir del número number pasado por parámetro
number Simplemente, el número en cuestión. Notación preferida 
            
                //Notación literal
                const number = 2;
                const decimal = 46.5;
                cons legibleNumber = 5_000_000;

Los números decimales los separamos con un punto (.) y de forma opcional, podemos utilizar guión bajo (_) para separar visualmente y reconocer las magnitudes que usamos.

Existe una serie de constantes definidas que marcan los límites mínimo o máximo.

Constante Valor Descripción
Numbre.MAX_VALUE ~21024 Valor más grande
Number.MIN_VALUE ~ 5x10-324 Valor más pequeño
Number.MAX_SAFE_INTEGER 253-1 Valor seguro más grande
Number.MIN_SAFE_INTEGER -(253-1) Valor seguro más pequeño
Number.EPSILON 2-52 Número muy pequeño

El valor Number.EPSILON se suele utilizar para representar cantidades matemáticas muy pequeñas. Number.MAX_VALUE es el valor máximo que es porsible representar y Number.MAX_SAFE_INTEGER es el valor máximo para realizar cálculos con seguridad.

Valores infinitos

Si quieres representar un número muy alto, sin importar el valor en sí, puedes utilizar la constante Infinity y -Infinity para valores muy bajos.

Constante Valor Descripción
Number.POSITIVE_INFINITY Infinity Infinito positivo
Number.NEGATIVE_INFINITY -Infinity Infinito negativo

Los valores infinitos se pueden operar con números, pero ten en cuenta las reglas de los infinitos, y que en algunos casos, puede resultar una indeterminación, por lo que nos dará como resultado NaN (not a number).

comprobaciones numéricas

Podemos comprobar con uno de los métodos estáticos de Number que tenemos a continuación

Método Descripción
Number.isFinite(number) Comprueba si number es un número finito
Number.isInteger(number) Comprueba si number es un número entero
Number.isSafeInteger(number) Comprueba si number es un número seguro

Estas funciones devuelven un boolean, que son ideales para usarlas como condiciones en bucles o condicionales.

            
                //Número finito
                Number.isFinite(56);        //true
                Number.isFinite(566.2);     //true
                Number.isFinite(Infinity);  //false, es infinito

                //Número entero
                Number.isInteger(8);        //true
                Number.isInteger(8.4);      //false, es decimal

                //Número seguro
                Number.isSafeInteger(1e15)  //true
                Number.isSafeInteger(1e16)  //false

                1e309 === Infinity;         //true
                Number.isSafeInteger(1e309) //false

Representación numérica

Tenemos dos formas de representar los números: la notación exponencial (basada en potencias de 10) y la notación de punto fijo (una cantidad de dígitos para la parte entera, y otra para la parte decimal). Se puede cambiar entre ellas utilizando los métodos:

Método Descripción
.toExponential(digits) Convierte el número a notación exponencial con digits decimales
.toFixed(digits) Convierte el número a notación de punto fijo con digits decimales
.toPrecisión(size) Utiliza size dígitos de precisión en el número

El método .toPrecision() puede cambiar la precisión de un número.

            
                //Notación exponencial
                (1.25).toExponential(0);    //"1e+0"
                (1.25).toExponential(1);    //"1.3e+0"
                (1.25).toExponential(2);    //"1.25e+0"
                (1.25).toExponential(3);    //"1.250e+0"

                //Notación punto fijo
                (1.25).toFixed(0);          //"1"
                (1.25).toFixed(1);          //"1.3"
                (1.25).toFixed(2);          //"1.25"
                (1.25).toFixed(3);          //"1.250"

                //Cambiando precisión
                (523.75).toPrecisión(1);    //"5e+2"
                (523.75).toPrecisión(2);    //"5.2e+2"
                (523.75).toPrecisión(3);    //"524"
                (523.75).toPrecisión(4);    //"523.8"
                (523.75).toPrecisión(5);    //"523.75"

En todos los casos, el número se convierte a tipo de dato string.
